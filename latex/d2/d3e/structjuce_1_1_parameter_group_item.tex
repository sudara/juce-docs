Creates a component that will be used to represent this item.

You don\textquotesingle{}t have to implement this method -\/ if it returns nullptr then no component will be used for the item, and you can just draw it using the \mbox{\hyperlink{class_tree_view_item_a31587d447a9622b647ac9d8d6b9e63be}{paint\+Item()}} callback. But if you do return a component, it will be positioned in the \mbox{\hyperlink{class_tree_view}{Tree\+View}} so that it can be used to represent this item.

The component returned will be managed by the \mbox{\hyperlink{class_tree_view}{Tree\+View}} and will be deleted later when it goes off the screen or is no longer needed. Its position and size will be completely managed by the tree, so don\textquotesingle{}t attempt to move it around.

Something you may want to do with your component is to give it a pointer to the \mbox{\hyperlink{class_tree_view}{Tree\+View}} that created it. This is perfectly safe, and there\textquotesingle{}s no danger of it becoming a dangling pointer because the \mbox{\hyperlink{class_tree_view}{Tree\+View}} will always delete the component before it is itself deleted.

As long as you stick to these rules you can return whatever kind of component you like. It\textquotesingle{}s most useful if you\textquotesingle{}re doing things like drag-\/and-\/drop of items, or want to use a \mbox{\hyperlink{class_label}{Label}} component to edit item names, etc. 

Reimplemented from \mbox{\hyperlink{class_tree_view_item_a7234dd675142d729485b3e66b33fc832}{Tree\+View\+Item}}.

Tells the tree whether this item can potentially be opened.

If your item could contain sub-\/items, this should return true; if it returns false then the tree will not try to open the item. This determines whether or not the item will be drawn with a \textquotesingle{}plus\textquotesingle{} button next to it.
