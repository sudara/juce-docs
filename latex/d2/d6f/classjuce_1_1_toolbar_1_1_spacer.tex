This method must return the size criteria for this item, based on a given toolbar size and orientation.

The preferred\+Size, min\+Size and max\+Size values must all be set by your implementation method. If the toolbar is horizontal, these will be the width of the item; for a vertical toolbar, they refer to the item\textquotesingle{}s height.

The preferred\+Size is the size that the component would like to be, and this must be between the min and max sizes. For a fixed-\/size item, simply set all three variables to the same value.

The toolbar\+Thickness parameter tells you the depth of the toolbar -\/ the same as calling \mbox{\hyperlink{class_toolbar_a2b99b5c21d3d282b1530443473c45c23}{Toolbar\+::get\+Thickness()}}.

The is\+Toolbar\+Vertical parameter tells you whether the bar is oriented horizontally or vertically.

Components can override this method to draw their content.

The \mbox{\hyperlink{classjuce_1_1_toolbar_1_1_spacer_a41a7f52fbad855dafeec29c05ec11213}{paint()}} method gets called when a region of a component needs redrawing, either because the component\textquotesingle{}s \mbox{\hyperlink{class_component_af3ab3eabec93dd60faf983370f3a1206}{repaint()}} method has been called, or because something has happened on the screen that means a section of a window needs to be redrawn.

Any child components will draw themselves over whatever this method draws. If you need to paint over the top of your child components, you can also implement the \mbox{\hyperlink{class_component_aa15dc77c06b5a0a4192090b4a323f79c}{paint\+Over\+Children()}} method to do this.

If you want to cause a component to redraw itself, this is done asynchronously -\/ calling the \mbox{\hyperlink{class_component_af3ab3eabec93dd60faf983370f3a1206}{repaint()}} method marks a region of the component as \char`\"{}dirty\char`\"{}, and the \mbox{\hyperlink{classjuce_1_1_toolbar_1_1_spacer_a41a7f52fbad855dafeec29c05ec11213}{paint()}} method will automatically be called sometime later, by the message thread, to paint any bits that need refreshing. In JUCE (and almost all modern UI frameworks), you never redraw something synchronously.

You should never need to call this method directly -\/ to take a snapshot of the component you could use \mbox{\hyperlink{class_component_a275e46e5a8147f89c142c91ba5c793a5}{create\+Component\+Snapshot()}} or \mbox{\hyperlink{class_component_a2c960bd5ec5ac76d18e82b6883096714}{paint\+Entire\+Component()}}.

Your subclass should use this method to draw its content area.

If the component you\textquotesingle{}re writing isn\textquotesingle{}t a button, you can just do nothing in this method.
