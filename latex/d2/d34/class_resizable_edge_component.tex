\doxysubsection{Detailed Description}
A component that resizes its parent component when dragged.

This component forms a bar along one edge of a component, allowing it to be dragged by that edges to resize it.

To use it, just add it to your component, positioning it along the appropriate edge. Make sure you reposition the resizer component each time the parent\textquotesingle{}s size changes, to keep it in the correct position.

Creates a resizer bar.

Pass in the target component which you want to be resized when this one is dragged. The target component will usually be this component\textquotesingle{}s parent, but this isn\textquotesingle{}t mandatory.

Remember that when the target component is resized, it\textquotesingle{}ll need to move and resize this component to keep it in place, as this won\textquotesingle{}t happen automatically.

If the constrainer parameter is not a nullptr, then this object will be used to enforce limits on the size and position that the component can be stretched to. Make sure that the constrainer isn\textquotesingle{}t deleted while still in use by this object.

{\footnotesize\ttfamily \mbox{\hyperlink{png_8h_aa8c59027f9ab2769342f248709d68d17}{void}} Resizable\+Edge\+Component\+::paint (\begin{DoxyParamCaption}\item[{\mbox{\hyperlink{class_graphics}{Graphics}} \&}]{g }\end{DoxyParamCaption})\hspace{0.3cm}{\ttfamily [override]}, {\ttfamily [protected]}, {\ttfamily [virtual]}}

Components can override this method to draw their content.

The \mbox{\hyperlink{class_resizable_edge_component_a7c070b054bd93f89caf72163a2bc18a7}{paint()}} method gets called when a region of a component needs redrawing, either because the component\textquotesingle{}s \mbox{\hyperlink{class_component_af3ab3eabec93dd60faf983370f3a1206}{repaint()}} method has been called, or because something has happened on the screen that means a section of a window needs to be redrawn.

Any child components will draw themselves over whatever this method draws. If you need to paint over the top of your child components, you can also implement the \mbox{\hyperlink{class_component_aa15dc77c06b5a0a4192090b4a323f79c}{paint\+Over\+Children()}} method to do this.

If you want to cause a component to redraw itself, this is done asynchronously -\/ calling the \mbox{\hyperlink{class_component_af3ab3eabec93dd60faf983370f3a1206}{repaint()}} method marks a region of the component as \char`\"{}dirty\char`\"{}, and the \mbox{\hyperlink{class_resizable_edge_component_a7c070b054bd93f89caf72163a2bc18a7}{paint()}} method will automatically be called sometime later, by the message thread, to paint any bits that need refreshing. In JUCE (and almost all modern UI frameworks), you never redraw something synchronously.

You should never need to call this method directly -\/ to take a snapshot of the component you could use \mbox{\hyperlink{class_component_a275e46e5a8147f89c142c91ba5c793a5}{create\+Component\+Snapshot()}} or \mbox{\hyperlink{class_component_a2c960bd5ec5ac76d18e82b6883096714}{paint\+Entire\+Component()}}.
