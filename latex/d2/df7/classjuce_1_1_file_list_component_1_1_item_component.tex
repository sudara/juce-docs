Components can override this method to draw their content.

The \mbox{\hyperlink{classjuce_1_1_file_list_component_1_1_item_component_ab002afc248a2aae80cd9bb3361fc29e6}{paint()}} method gets called when a region of a component needs redrawing, either because the component\textquotesingle{}s \mbox{\hyperlink{class_component_af3ab3eabec93dd60faf983370f3a1206}{repaint()}} method has been called, or because something has happened on the screen that means a section of a window needs to be redrawn.

Any child components will draw themselves over whatever this method draws. If you need to paint over the top of your child components, you can also implement the \mbox{\hyperlink{class_component_aa15dc77c06b5a0a4192090b4a323f79c}{paint\+Over\+Children()}} method to do this.

If you want to cause a component to redraw itself, this is done asynchronously -\/ calling the \mbox{\hyperlink{class_component_af3ab3eabec93dd60faf983370f3a1206}{repaint()}} method marks a region of the component as \char`\"{}dirty\char`\"{}, and the \mbox{\hyperlink{classjuce_1_1_file_list_component_1_1_item_component_ab002afc248a2aae80cd9bb3361fc29e6}{paint()}} method will automatically be called sometime later, by the message thread, to paint any bits that need refreshing. In JUCE (and almost all modern UI frameworks), you never redraw something synchronously.

You should never need to call this method directly -\/ to take a snapshot of the component you could use \mbox{\hyperlink{class_component_a275e46e5a8147f89c142c91ba5c793a5}{create\+Component\+Snapshot()}} or \mbox{\hyperlink{class_component_a2c960bd5ec5ac76d18e82b6883096714}{paint\+Entire\+Component()}}.

Called back by a \mbox{\hyperlink{class_time_slice_thread}{Time\+Slice\+Thread}}.

When you register this class with it, a \mbox{\hyperlink{class_time_slice_thread}{Time\+Slice\+Thread}} will repeatedly call this method.

The implementation of this method should use its time-\/slice to do something that\textquotesingle{}s quick -\/ never block for longer than absolutely necessary.

\begin{DoxyReturn}{Returns}
Your method should return the number of milliseconds which it would like to wait before being called again. Returning 0 will make the thread call again as soon as possible (after possibly servicing other busy clients). If you return a value below zero, your client will be removed from the list of clients, and won\textquotesingle{}t be called again. The value you specify isn\textquotesingle{}t a guarantee, and is only used as a hint by the thread -\/ the actual time before the next callback may be more or less than specified. You can force the \mbox{\hyperlink{class_time_slice_thread}{Time\+Slice\+Thread}} to wake up and poll again immediately by calling its notify() method. 
\end{DoxyReturn}
