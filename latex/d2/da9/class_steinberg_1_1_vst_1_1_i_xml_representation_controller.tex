A representation based on XML is a way to export, structure, and group plug-\/ins parameters for a specific remote (hardware or software rack (such as quick controls)). ~\newline
It allows to describe each parameter more precisely (what is the best matching to a knob, different title lengths matching limited remote display,...).~\newline
 See an \mbox{\hyperlink{class_steinberg_1_1_vst_1_1_i_xml_representation_controller_Example}{Example}}. ~\newline
~\newline

\begin{DoxyItemize}
\item A representation is composed of pages (this means that to see all exported parameters, the user has to navigate through the pages).
\item A page is composed of cells (for example 8 cells per page).
\item A cell is composed of layers (for example a cell could have a knob, a display, and a button, which means 3 layers).
\item A layer is associated to a plug-\/in parameter using the Parameter\+ID as identifier\+:
\begin{DoxyItemize}
\item it could be a knob with a display for title and/or value, this display uses the same parameter\+Id, but it could an another one.
\item switch
\item link which allows to jump directly to a subpage (another page)
\item more... See \mbox{\hyperlink{namespace_steinberg_1_1_vst_1_1_layer_type}{Vst\+::\+Layer\+Type}}
\end{DoxyItemize}
\end{DoxyItemize}~\newline
