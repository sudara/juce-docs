{\em dest\+Channels} & an array of buffers into which the sample data for each channel will be written. If it is a floating-\/point format, you should cast the resulting array to a (float$\ast$$\ast$) to get the values (in the range -\/1.\+0 to 1.\+0 or beyond) If the format is stereo, then dest\+Channels\mbox{[}0\mbox{]} is the left channel data, and dest\+Channels\mbox{[}1\mbox{]} is the right channel. E.\+g. if you\textquotesingle{}re reading a mono file and you pass 2 channels to this method, then if fill\+Leftover\+Channels\+With\+Copies is true, both destination channels will be filled with the same data from the file\textquotesingle{}s single channel. If fill\+Leftover\+Channels\+With\+Copies was false, then only the first channel would be filled with the file\textquotesingle{}s contents, and the second would be cleared. Scans the source looking for a sample whose magnitude is in a specified range.

This will read from the source, either forwards or backwards between two sample positions, until it finds a sample whose magnitude lies between two specified levels.

If it finds a suitable sample, it returns its position; if not, it will return -\/1.

There\textquotesingle{}s also a minimum\+Consecutive\+Samples setting to help avoid spikes or zero-\/crossing points when you\textquotesingle{}re searching for a continuous range of samples
