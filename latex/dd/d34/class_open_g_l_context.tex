\doxysubsection{Detailed Description}
Creates an Open\+GL context, which can be attached to a component.

To render some Open\+GL, you should create an instance of an \mbox{\hyperlink{class_open_g_l_context}{Open\+GLContext}}, and call \mbox{\hyperlink{class_open_g_l_context_ad320cd274bba96c11dc1ea32310ad37c}{attach\+To()}} to make it use a component as its render target.

To provide threaded rendering, you can supply an \mbox{\hyperlink{class_open_g_l_renderer}{Open\+GLRenderer}} object that will be used to render each frame.

Before your target component or \mbox{\hyperlink{class_open_g_l_renderer}{Open\+GLRenderer}} is deleted, you MUST call \mbox{\hyperlink{class_open_g_l_context_a7504aa8928c854732090ee9583e9f37d}{detach()}} or delete the \mbox{\hyperlink{class_open_g_l_context}{Open\+GLContext}} to allow the background thread to stop and the native resources to be freed safely.

Attaches the context to a target component.

If the component is not fully visible, this call will wait until the component is shown before actually creating a native context for it.

When a native context is created, a thread is started, and will be used to call the \mbox{\hyperlink{class_open_g_l_renderer}{Open\+GLRenderer}} methods. The context will be floated above the target component, and when the target moves, it will track it. If the component is hidden/shown, the context may be deleted and re-\/created. \mbox{\Hypertarget{class_open_g_l_context_ac0bf3594cb79029af5ee5eb66a14f152}\label{class_open_g_l_context_ac0bf3594cb79029af5ee5eb66a14f152}}
