Sets the item that is displayed in the \mbox{\hyperlink{class_tree_view}{Tree\+View}}.

A tree has a single root item which contains as many sub-\/items as it needs. If you want the tree to contain a number of root items, you should still use a single root item above these, but hide it using \mbox{\hyperlink{class_tree_view_a3c28114193951c8079fdfeb95738f410}{set\+Root\+Item\+Visible()}}.

You can pass nullptr to this method to clear the tree and remove its current root item.

The object passed in will not be deleted by the \mbox{\hyperlink{class_tree_view}{Tree\+View}}, it\textquotesingle{}s up to the caller to delete it when no longer needed. BUT make absolutely sure that you don\textquotesingle{}t delete this item until you\textquotesingle{}ve removed it from the tree, either by calling set\+Root\+Item (nullptr), or by deleting the tree first. You can also use \mbox{\hyperlink{class_tree_view_ad5245e71c0684de6cf942238da54b9cc}{delete\+Root\+Item()}} as a quick way to delete it. \mbox{\Hypertarget{class_tree_view_a3c28114193951c8079fdfeb95738f410}\label{class_tree_view_a3c28114193951c8079fdfeb95738f410}}

Changes whether the tree\textquotesingle{}s root item is shown or not.

If the root item is hidden, only its sub-\/items will be shown in the \mbox{\hyperlink{class_tree_view}{Tree\+View}} -\/ this lets you make the tree look as if it\textquotesingle{}s got many root items. If it\textquotesingle{}s hidden, this call will also make sure the root item is open (otherwise the \mbox{\hyperlink{class_tree_view}{Tree\+View}} would look empty).
