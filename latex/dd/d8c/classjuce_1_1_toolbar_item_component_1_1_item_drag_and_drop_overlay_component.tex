Called when a mouse button is pressed.

The \mbox{\hyperlink{class_mouse_event}{Mouse\+Event}} object passed in contains lots of methods for finding out which button was pressed, as well as which modifier keys (e.\+g. shift, ctrl) were held down at the time.

Once a button is held down, the mouse\+Drag method will be called when the mouse moves, until the button is released.

{\footnotesize\ttfamily \mbox{\hyperlink{png_8h_aa8c59027f9ab2769342f248709d68d17}{void}} juce\+::\+Toolbar\+Item\+Component\+::\+Item\+Drag\+And\+Drop\+Overlay\+Component\+::paint (\begin{DoxyParamCaption}\item[{\mbox{\hyperlink{class_graphics}{Graphics}} \&}]{g }\end{DoxyParamCaption})\hspace{0.3cm}{\ttfamily [inline]}, {\ttfamily [override]}, {\ttfamily [virtual]}}

Components can override this method to draw their content.

The \mbox{\hyperlink{classjuce_1_1_toolbar_item_component_1_1_item_drag_and_drop_overlay_component_a75f8d9ef2235f9af5a2a26703c3b3ae5}{paint()}} method gets called when a region of a component needs redrawing, either because the component\textquotesingle{}s \mbox{\hyperlink{class_component_af3ab3eabec93dd60faf983370f3a1206}{repaint()}} method has been called, or because something has happened on the screen that means a section of a window needs to be redrawn.

Any child components will draw themselves over whatever this method draws. If you need to paint over the top of your child components, you can also implement the \mbox{\hyperlink{class_component_aa15dc77c06b5a0a4192090b4a323f79c}{paint\+Over\+Children()}} method to do this.

If you want to cause a component to redraw itself, this is done asynchronously -\/ calling the \mbox{\hyperlink{class_component_af3ab3eabec93dd60faf983370f3a1206}{repaint()}} method marks a region of the component as \char`\"{}dirty\char`\"{}, and the \mbox{\hyperlink{classjuce_1_1_toolbar_item_component_1_1_item_drag_and_drop_overlay_component_a75f8d9ef2235f9af5a2a26703c3b3ae5}{paint()}} method will automatically be called sometime later, by the message thread, to paint any bits that need refreshing. In JUCE (and almost all modern UI frameworks), you never redraw something synchronously.

You should never need to call this method directly -\/ to take a snapshot of the component you could use \mbox{\hyperlink{class_component_a275e46e5a8147f89c142c91ba5c793a5}{create\+Component\+Snapshot()}} or \mbox{\hyperlink{class_component_a2c960bd5ec5ac76d18e82b6883096714}{paint\+Entire\+Component()}}.
