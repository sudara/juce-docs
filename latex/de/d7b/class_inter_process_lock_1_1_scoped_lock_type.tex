\doxysubsection{Detailed Description}
Automatically locks and unlocks an \mbox{\hyperlink{class_inter_process_lock}{Inter\+Process\+Lock}} object.

This works like a Scoped\+Lock, but using an Interprocess\+Lock rather than a \mbox{\hyperlink{class_critical_section}{Critical\+Section}}.

Creates a scoped lock.

As soon as it is created, this will lock the \mbox{\hyperlink{class_inter_process_lock}{Inter\+Process\+Lock}}, and when the \mbox{\hyperlink{class_inter_process_lock_1_1_scoped_lock_type}{Scoped\+Lock\+Type}} object is deleted, the \mbox{\hyperlink{class_inter_process_lock}{Inter\+Process\+Lock}} will be unlocked.

Note that since an Interprocess\+Lock can fail due to errors, you should check \mbox{\hyperlink{class_inter_process_lock_1_1_scoped_lock_type_ab4c0eda7e9ab3196721f5707e36f4b4a}{is\+Locked()}} to make sure that the lock was successful before using it.

Make sure this object is created and deleted by the same thread, otherwise there are no guarantees what will happen! Best just to use it as a local stack object, rather than creating one with the new() operator. \mbox{\Hypertarget{class_inter_process_lock_1_1_scoped_lock_type_a2517d70321d5acabdfe5a96f11ef9c8f}\label{class_inter_process_lock_1_1_scoped_lock_type_a2517d70321d5acabdfe5a96f11ef9c8f}}
