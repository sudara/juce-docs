\doxysubsection{Detailed Description}
A re-\/entrant mutex.

A \mbox{\hyperlink{class_critical_section}{Critical\+Section}} acts as a re-\/entrant mutex object. The best way to lock and unlock one of these is by using RAII in the form of a local Scoped\+Lock object -\/ have a look through the codebase for many examples of how to do this.

In almost all cases you\textquotesingle{}ll want to declare your \mbox{\hyperlink{class_critical_section}{Critical\+Section}} as a member variable. Occasionally you may want to declare one as a static variable, but in that case the usual C++ static object order-\/of-\/construction warnings should be heeded.

{\footnotesize\ttfamily \mbox{\hyperlink{png_8h_aa8c59027f9ab2769342f248709d68d17}{void}} Critical\+Section\+::enter (\begin{DoxyParamCaption}{ }\end{DoxyParamCaption}) const\hspace{0.3cm}{\ttfamily [noexcept]}}

Acquires the lock.

If the lock is already held by the caller thread, the method returns immediately. If the lock is currently held by another thread, this will wait until it becomes free.

It\textquotesingle{}s strongly recommended that you never call this method directly -\/ instead use the Scoped\+Lock class to manage the locking using an RAII pattern instead.
