\doxysubsection{Detailed Description}
Allows threads to wait for events triggered by other threads.

A thread can call \mbox{\hyperlink{class_waitable_event_a1e5eb0e88e9e88aa34b58395d2bff1e0}{Waitable\+Event\+::wait()}} to suspend the calling thread until another thread wakes it up by calling the \mbox{\hyperlink{class_waitable_event_a7e51d16c549d2c9c200121161f859d5e}{Waitable\+Event\+::signal()}} method.

Wakes up any threads that are currently waiting on this object.

If \mbox{\hyperlink{class_waitable_event_a7e51d16c549d2c9c200121161f859d5e}{signal()}} is called when nothing is waiting, the next thread to call \mbox{\hyperlink{class_waitable_event_a1e5eb0e88e9e88aa34b58395d2bff1e0}{wait()}} will return immediately and reset the signal.

If the \mbox{\hyperlink{class_waitable_event}{Waitable\+Event}} is manual reset, all current and future threads that wait upon this object will be woken, until \mbox{\hyperlink{class_waitable_event_a57ae80211fcb98ba75f2c57be581ff75}{reset()}} is explicitly called.

If the \mbox{\hyperlink{class_waitable_event}{Waitable\+Event}} is automatic reset, and one or more threads is waiting upon the object, then one of them will be woken up. If no threads are currently waiting, then the next thread to call \mbox{\hyperlink{class_waitable_event_a1e5eb0e88e9e88aa34b58395d2bff1e0}{wait()}} will be woken up. As soon as a thread is woken, the signal is automatically reset.

Suspends the calling thread until the event has been signalled.

This will wait until the object\textquotesingle{}s \mbox{\hyperlink{class_waitable_event_a7e51d16c549d2c9c200121161f859d5e}{signal()}} method is called by another thread, or until the timeout expires.

After the event has been signalled, this method will return true and if manual\+Reset was set to false in the \mbox{\hyperlink{class_waitable_event}{Waitable\+Event}}\textquotesingle{}s constructor, then the event will be reset.
