The \mbox{\hyperlink{classjuce_1_1_audio_device_settings_panel_1_1_channel_selector_list_box_a9be714a0cdd42e2cba0c31787bed873a}{paint()}} method gets called when a region of a component needs redrawing, either because the component\textquotesingle{}s \mbox{\hyperlink{class_component_af3ab3eabec93dd60faf983370f3a1206}{repaint()}} method has been called, or because something has happened on the screen that means a section of a window needs to be redrawn.

Any child components will draw themselves over whatever this method draws. If you need to paint over the top of your child components, you can also implement the \mbox{\hyperlink{class_list_box_a5797c000ce38f73c2d45e0bf0e283141}{paint\+Over\+Children()}} method to do this.

If you want to cause a component to redraw itself, this is done asynchronously -\/ calling the \mbox{\hyperlink{class_component_af3ab3eabec93dd60faf983370f3a1206}{repaint()}} method marks a region of the component as \char`\"{}dirty\char`\"{}, and the \mbox{\hyperlink{classjuce_1_1_audio_device_settings_panel_1_1_channel_selector_list_box_a9be714a0cdd42e2cba0c31787bed873a}{paint()}} method will automatically be called sometime later, by the message thread, to paint any bits that need refreshing. In JUCE (and almost all modern UI frameworks), you never redraw something synchronously.

You should never need to call this method directly -\/ to take a snapshot of the component you could use \mbox{\hyperlink{class_component_a275e46e5a8147f89c142c91ba5c793a5}{create\+Component\+Snapshot()}} or \mbox{\hyperlink{class_component_a2c960bd5ec5ac76d18e82b6883096714}{paint\+Entire\+Component()}}.
