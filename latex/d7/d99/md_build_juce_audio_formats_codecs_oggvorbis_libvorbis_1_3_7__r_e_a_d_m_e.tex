Vorbis is a general purpose audio and music encoding format contemporary to MPEG-\/4\textquotesingle{}s AAC and Twin\+VQ, the next generation beyond MPEG audio layer 3. Unlike the MPEG sponsored formats (and other proprietary formats such as Real\+Audio G2 and Windows\textquotesingle{} flavor of the month), the Vorbis CODEC specification belongs to the public domain. All the technical details are published and documented, and any software entity may make full use of the format without license fee, royalty or patent concerns.

This package contains\+:


\begin{DoxyItemize}
\item libvorbis, a BSD-\/style license software implementation of the Vorbis specification by the Xiph.\+Org Foundation (\href{https://xiph.org/}{\texttt{ https\+://xiph.\+org/}})
\item libvorbisfile, a BSD-\/style license convenience library built on Vorbis designed to simplify common uses
\item libvorbisenc, a BSD-\/style license library that provides a simple, programmatic encoding setup interface
\item example code making use of libogg, libvorbis, libvorbisfile and libvorbisenc
\end{DoxyItemize}\hypertarget{md_build_juce_audio_formats_codecs_oggvorbis_libvorbis_1_3_7__r_e_a_d_m_e_autotoc_md38}{}\doxysection{What\textquotesingle{}s here}\label{md_build_juce_audio_formats_codecs_oggvorbis_libvorbis_1_3_7__r_e_a_d_m_e_autotoc_md38}
This source distribution includes libvorbis and an example encoder/player to demonstrate use of libvorbis as well as documentation on the Ogg Vorbis audio coding format.

You\textquotesingle{}ll need libogg (distributed separately) to compile this library. A more comprehensive set of utilities is available in the vorbis-\/tools package.

Ogg supports building using \href{https://cmake.org/}{\texttt{ CMake}}. CMake is a meta build system that generates native projects for each platform. To generate projects just run cmake replacing {\ttfamily YOUR-\/\+PROJECT-\/\+GENERATOR} with a proper generator from a list \href{https://cmake.org/cmake/help/latest/manual/cmake-generators.7.html}{\texttt{ here}}\+: \begin{DoxyVerb}cmake -G YOUR-PROJECT-GENERATOR .
\end{DoxyVerb}
 Note that by default cmake generates projects that will build static libraries. To generate projects that will build dynamic library use {\ttfamily BUILD\+\_\+\+SHARED\+\_\+\+LIBS} option like this\+: \begin{DoxyVerb}cmake -G YOUR-PROJECT-GENERATOR -DBUILD_SHARED_LIBS=1 .
\end{DoxyVerb}
 After projects are generated use them as usual\hypertarget{md_build_juce_audio_formats_codecs_oggvorbis_libvorbis_1_3_7__r_e_a_d_m_e_autotoc_md45}{}\doxysubsubsection{Building on Windows}\label{md_build_juce_audio_formats_codecs_oggvorbis_libvorbis_1_3_7__r_e_a_d_m_e_autotoc_md45}
