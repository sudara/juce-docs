\doxysubsection{Detailed Description}
On API 16 to 26 Open\+SL ES will be used. When using Open\+SL ES the optimal values for sample\+Rate and frames\+Per\+Burst are not known by the native code. On API 17+ these values should be obtained from the Audio\+Manager using this code\+:


\begin{DoxyPre}{\ttfamily 
// Note that this technique only works for built-\/in speakers and headphones.
AudioManager myAudioMgr = (AudioManager) getSystemService(Context.AUDIO\_SERVICE);
\mbox{\hyperlink{class_string}{String}} sampleRateStr = myAudioMgr.getProperty(AudioManager.PROPERTY\_OUTPUT\_SAMPLE\_RATE);
int defaultSampleRate = Integer.parseInt(sampleRateStr);
\mbox{\hyperlink{class_string}{String}} framesPerBurstStr = myAudioMgr.getProperty(AudioManager.PROPERTY\_OUTPUT\_FRAMES\_PER\_BUFFER);
int defaultFramesPerBurst = Integer.parseInt(framesPerBurstStr);
}\end{DoxyPre}


It can then be passed down to Oboe through JNI.

AAudio will get the optimal frames\+Per\+Burst from the HAL and will ignore this value.

The default sample rate to use when opening new audio streams

The following default values are used when oboe does not have any better way of determining the optimal values for an audio stream. This can happen when\+:


\begin{DoxyItemize}
\item Client is creating a stream on API \texorpdfstring{$<$}{<} 26 (Open\+SLES) but has not supplied the optimal sample rate and/or frames per burst
\item Client is creating a stream on API 16 (Open\+SLES) where Audio\+Manager.\+PROPERTY\+\_\+\+OUTPUT\+\_\+$\ast$ values are not available 
\end{DoxyItemize}
