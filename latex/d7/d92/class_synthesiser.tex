\doxysubsection{Detailed Description}
Base class for a musical device that can play sounds.

To create a synthesiser, you\textquotesingle{}ll need to create a subclass of \mbox{\hyperlink{class_synthesiser_sound}{Synthesiser\+Sound}} to describe each sound available to your synth, and a subclass of \mbox{\hyperlink{class_synthesiser_voice}{Synthesiser\+Voice}} which can play back one of these sounds.

Then you can use the \mbox{\hyperlink{class_synthesiser_a24e5413809d05be0c2b94e584d198801}{add\+Voice()}} and \mbox{\hyperlink{class_synthesiser_a019093cdd2ec928058c261fcb1cd7e72}{add\+Sound()}} methods to give the synthesiser a set of sounds, and a set of voices it can use to play them. If you only give it one voice it will be monophonic -\/ the more voices it has, the more polyphony it\textquotesingle{}ll have available.

Then repeatedly call the \mbox{\hyperlink{class_synthesiser_a56b85b35ec0fbf296c58571ab5bc2ed8}{render\+Next\+Block()}} method to produce the audio. Any midi events that go in will be scanned for note on/off messages, and these are used to start and stop the voices playing the appropriate sounds.

While it\textquotesingle{}s playing, you can also cause notes to be triggered by calling the \mbox{\hyperlink{class_synthesiser_a4d9ef4711cb0b2c62fabceb4d354853c}{note\+On()}}, \mbox{\hyperlink{class_synthesiser_a2f26dde8ce8873554edf6a9d06f168c9}{note\+Off()}} and other controller methods.

Before rendering, be sure to call the \mbox{\hyperlink{class_synthesiser_a4a7d7d7efeb1ad90488d36c35c530f10}{set\+Current\+Playback\+Sample\+Rate()}} to tell it what the target playback rate is. This value is passed on to the voices so that they can pitch their output correctly.

{\footnotesize\ttfamily \mbox{\hyperlink{png_8h_aa8c59027f9ab2769342f248709d68d17}{void}} Synthesiser\+::all\+Notes\+Off (\begin{DoxyParamCaption}\item[{\mbox{\hyperlink{group__juce__audio__plugin__client-_unity_ga966b05a42d98c7296097e073c627dcd0}{int}}}]{midi\+Channel,  }\item[{bool}]{allow\+Tail\+Off }\end{DoxyParamCaption})\hspace{0.3cm}{\ttfamily [virtual]}}

Turns off all notes.

This will turn off any voices that are playing a sound on the given midi channel.

If midi\+Channel is 0 or less, then all voices will be turned off, regardless of which channel they\textquotesingle{}re playing. Otherwise it represents a valid midi channel, from 1 to 16 inclusive.

If allow\+Tail\+Off is true, the voices will be allowed to fade out the notes gracefully (if they can do). If this is false, the notes will all be cut off immediately.

This will process the next num\+Samples of data from all the voices, and add that output to the audio block supplied, starting from the offset specified. Note that the data will be added to the current contents of the buffer, so you should clear it before calling this method if necessary.

Sets a minimum limit on the size to which audio sub-\/blocks will be divided when rendering.

When rendering, the audio blocks that are passed into \mbox{\hyperlink{class_synthesiser_a56b85b35ec0fbf296c58571ab5bc2ed8}{render\+Next\+Block()}} will be split up into smaller blocks that lie between all the incoming midi messages, and it is these smaller sub-\/blocks that are rendered with multiple calls to \mbox{\hyperlink{class_synthesiser_a1367e7a903997355fcc5f1b875dd98f1}{render\+Voices()}}.

Obviously in a pathological case where there are midi messages on every sample, then \mbox{\hyperlink{class_synthesiser_a1367e7a903997355fcc5f1b875dd98f1}{render\+Voices()}} could be called once per sample and lead to poor performance, so this setting allows you to set a lower limit on the block size.

The default setting is 32, which means that midi messages are accurate to about \texorpdfstring{$<$}{<} 1ms accuracy, which is probably fine for most purposes, but you may want to increase or decrease this value for your synth.

If should\+Be\+Strict is true, the audio sub-\/blocks will strictly never be smaller than num\+Samples.

If should\+Be\+Strict is false (default), the first audio sub-\/block in the buffer is allowed to be smaller, to make sure that the first MIDI event in a buffer will always be sample-\/accurate (this can sometimes help to avoid quantisation or phasing issues). \mbox{\Hypertarget{class_synthesiser_ae0bbfe2c3819167dc5d98b27b21843eb}\label{class_synthesiser_ae0bbfe2c3819167dc5d98b27b21843eb}}
