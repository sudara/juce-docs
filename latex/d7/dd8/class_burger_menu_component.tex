\doxysubsection{Detailed Description}
A component which lists all menu items and groups them into categories by their respective parent menus. This kind of component is often used for so-\/called \char`\"{}burger\char`\"{} menus in mobile apps.

Called to let the component react to a change in the look-\/and-\/feel setting.

When the look-\/and-\/feel is changed for a component, this will be called in all its child components, recursively.

It can also be triggered manually by the \mbox{\hyperlink{class_component_ac314d2449213d7942eca083addfcd097}{send\+Look\+And\+Feel\+Change()}} method, in case an application uses a \mbox{\hyperlink{class_look_and_feel}{Look\+And\+Feel}} class that might have changed internally.

{\footnotesize\ttfamily \mbox{\hyperlink{png_8h_aa8c59027f9ab2769342f248709d68d17}{void}} Burger\+Menu\+Component\+::set\+Model (\begin{DoxyParamCaption}\item[{\mbox{\hyperlink{class_menu_bar_model}{Menu\+Bar\+Model}} $\ast$}]{new\+Model }\end{DoxyParamCaption})}

Changes the model object to use to control the burger menu.

This can be a nullptr, in which case the bar will be empty. This object will not be owned by the \mbox{\hyperlink{class_burger_menu_component}{Burger\+Menu\+Component}} so it is up to you to manage its lifetime. Don\textquotesingle{}t delete the object that is passed-\/in while it\textquotesingle{}s still being used by this Menu\+Bar. Any submenus in your \mbox{\hyperlink{class_menu_bar_model}{Menu\+Bar\+Model}} will be recursively flattened and added to the top-\/level burger menu section.
