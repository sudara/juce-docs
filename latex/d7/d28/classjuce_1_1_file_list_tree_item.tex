Called when an item is opened or closed.

When \mbox{\hyperlink{class_tree_view_item_ae1d86e63026471f8edea97f7dc5fd05f}{set\+Open()}} is called and the item has specified that it might have sub-\/items with the \mbox{\hyperlink{classjuce_1_1_file_list_tree_item_ac90b3f3df0543f158e77165c99101499}{might\+Contain\+Sub\+Items()}} method, this method is called to let the item create or manage its sub-\/items.

So when this is called with is\+Now\+Open set to true (i.\+e. when the item is being opened), a subclass might choose to use \mbox{\hyperlink{class_tree_view_item_a3e5f73cd812c6626047d38307560aaeb}{clear\+Sub\+Items()}} and \mbox{\hyperlink{class_tree_view_item_a536e1e4edaf0b0a472c485050d54eb5a}{add\+Sub\+Item()}} to refresh its sub-\/item list.

When this is called with is\+Now\+Open set to false, the subclass might want to use \mbox{\hyperlink{class_tree_view_item_a3e5f73cd812c6626047d38307560aaeb}{clear\+Sub\+Items()}} to save on space, or it might choose to leave them, depending on the nature of the tree.

You could also use this callback as a trigger to start a background process which asynchronously creates sub-\/items and adds them, if that\textquotesingle{}s more appropriate for the task in hand.

Called back by a \mbox{\hyperlink{class_time_slice_thread}{Time\+Slice\+Thread}}.

When you register this class with it, a \mbox{\hyperlink{class_time_slice_thread}{Time\+Slice\+Thread}} will repeatedly call this method.

The implementation of this method should use its time-\/slice to do something that\textquotesingle{}s quick -\/ never block for longer than absolutely necessary.

\begin{DoxyReturn}{Returns}
Your method should return the number of milliseconds which it would like to wait before being called again. Returning 0 will make the thread call again as soon as possible (after possibly servicing other busy clients). If you return a value below zero, your client will be removed from the list of clients, and won\textquotesingle{}t be called again. The value you specify isn\textquotesingle{}t a guarantee, and is only used as a hint by the thread -\/ the actual time before the next callback may be more or less than specified. You can force the \mbox{\hyperlink{class_time_slice_thread}{Time\+Slice\+Thread}} to wake up and poll again immediately by calling its notify() method. 
\end{DoxyReturn}
