Called repeatedly to fetch subsequent blocks of audio data.

After calling the \mbox{\hyperlink{class_i_i_r_filter_audio_source_a136fa8f34b9f2f82dd8de3547a00f6a2}{prepare\+To\+Play()}} method, this callback will be made each time the audio playback hardware (or whatever other destination the audio data is going to) needs another block of data.

It will generally be called on a high-\/priority system thread, or possibly even an interrupt, so be careful not to do too much work here, as that will cause audio glitches!

Tells the source to prepare for playing.

An \mbox{\hyperlink{class_audio_source}{Audio\+Source}} has two states\+: prepared and unprepared.

The \mbox{\hyperlink{class_i_i_r_filter_audio_source_a136fa8f34b9f2f82dd8de3547a00f6a2}{prepare\+To\+Play()}} method is guaranteed to be called at least once on an \textquotesingle{}unprepared\textquotesingle{} source to put it into a \textquotesingle{}prepared\textquotesingle{} state before any calls will be made to \mbox{\hyperlink{class_i_i_r_filter_audio_source_a30dd0315f98655cca03e578e115742de}{get\+Next\+Audio\+Block()}}. This callback allows the source to initialise any resources it might need when playing.

Once playback has finished, the \mbox{\hyperlink{class_i_i_r_filter_audio_source_a1a4939d75cdf79c7a1bdb5cd601e88aa}{release\+Resources()}} method is called to put the stream back into an \textquotesingle{}unprepared\textquotesingle{} state.

Note that this method could be called more than once in succession without a matching call to \mbox{\hyperlink{class_i_i_r_filter_audio_source_a1a4939d75cdf79c7a1bdb5cd601e88aa}{release\+Resources()}}, so make sure your code is robust and can handle that kind of situation.


\begin{DoxyParams}{Parameters}

This number may vary slightly, because it will be dependent on audio hardware callbacks, and these aren\textquotesingle{}t guaranteed to always use a constant block size, so the source should be able to cope with small variations. \\

Allows the source to release anything it no longer needs after playback has stopped.

This will be called when the source is no longer going to have its \mbox{\hyperlink{class_i_i_r_filter_audio_source_a30dd0315f98655cca03e578e115742de}{get\+Next\+Audio\+Block()}} method called, so it should release any spare memory, etc. that it might have allocated during the \mbox{\hyperlink{class_i_i_r_filter_audio_source_a136fa8f34b9f2f82dd8de3547a00f6a2}{prepare\+To\+Play()}} call.

Note that there\textquotesingle{}s no guarantee that \mbox{\hyperlink{class_i_i_r_filter_audio_source_a136fa8f34b9f2f82dd8de3547a00f6a2}{prepare\+To\+Play()}} will actually have been called before \mbox{\hyperlink{class_i_i_r_filter_audio_source_a1a4939d75cdf79c7a1bdb5cd601e88aa}{release\+Resources()}}, and it may be called more than once in succession, so make sure your code is robust and doesn\textquotesingle{}t make any assumptions about when it will be called.
