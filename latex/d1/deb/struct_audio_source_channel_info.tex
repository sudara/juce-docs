{\footnotesize\ttfamily \mbox{\hyperlink{class_audio_buffer}{Audio\+Buffer}}$<$float$>$$\ast$ Audio\+Source\+Channel\+Info\+::buffer}

The destination buffer to fill with audio data.

When the \mbox{\hyperlink{class_audio_source_a36fc2484ead8fb8dc03be57d09dd5735}{Audio\+Source\+::get\+Next\+Audio\+Block()}} method is called, the active section of this buffer should be filled with whatever output the source produces.

Only the samples specified by the start\+Sample and num\+Samples members of this structure should be affected by the call.

The contents of the buffer when it is passed to the \mbox{\hyperlink{class_audio_source_a36fc2484ead8fb8dc03be57d09dd5735}{Audio\+Source\+::get\+Next\+Audio\+Block()}} method can be treated as the input if the source is performing some kind of filter operation, but should be cleared if this is not the case -\/ the \mbox{\hyperlink{struct_audio_source_channel_info_a6246ca9f1f6ef86dab31fc0eeaacb7a3}{clear\+Active\+Buffer\+Region()}} is a handy way of doing this.

The number of channels in the buffer could be anything, so the \mbox{\hyperlink{class_audio_source}{Audio\+Source}} must cope with this in whatever way is appropriate for its function. \mbox{\Hypertarget{struct_audio_source_channel_info_a6669c809cf3b47afa94af63d4fa48101}\label{struct_audio_source_channel_info_a6669c809cf3b47afa94af63d4fa48101}}
