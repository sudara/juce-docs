\doxysubsection{Detailed Description}
A progress bar component.

To use this, just create one and make it visible. It\textquotesingle{}ll run its own timer to keep an eye on a variable that you give it, and will automatically redraw itself when the variable changes.

If using \mbox{\hyperlink{class_look_and_feel___v4}{Look\+And\+Feel\+\_\+\+V4}} a circular spinning progress bar will be drawn if the width and height of the \mbox{\hyperlink{class_progress_bar}{Progress\+Bar}} are equal, otherwise the standard, linear \mbox{\hyperlink{class_progress_bar}{Progress\+Bar}} will be drawn.

For an easy way of running a background task with a dialog box showing its progress, see the \mbox{\hyperlink{class_thread_with_progress_window}{Thread\+With\+Progress\+Window}} class.

{\footnotesize\ttfamily \mbox{\hyperlink{png_8h_aa8c59027f9ab2769342f248709d68d17}{void}} Progress\+Bar\+::paint (\begin{DoxyParamCaption}\item[{\mbox{\hyperlink{class_graphics}{Graphics}} \&}]{g }\end{DoxyParamCaption})\hspace{0.3cm}{\ttfamily [override]}, {\ttfamily [protected]}, {\ttfamily [virtual]}}

Components can override this method to draw their content.

The \mbox{\hyperlink{class_progress_bar_ac52be9d48db6804ad15677a786b817c8}{paint()}} method gets called when a region of a component needs redrawing, either because the component\textquotesingle{}s \mbox{\hyperlink{class_component_af3ab3eabec93dd60faf983370f3a1206}{repaint()}} method has been called, or because something has happened on the screen that means a section of a window needs to be redrawn.

Any child components will draw themselves over whatever this method draws. If you need to paint over the top of your child components, you can also implement the \mbox{\hyperlink{class_component_aa15dc77c06b5a0a4192090b4a323f79c}{paint\+Over\+Children()}} method to do this.

If you want to cause a component to redraw itself, this is done asynchronously -\/ calling the \mbox{\hyperlink{class_component_af3ab3eabec93dd60faf983370f3a1206}{repaint()}} method marks a region of the component as \char`\"{}dirty\char`\"{}, and the \mbox{\hyperlink{class_progress_bar_ac52be9d48db6804ad15677a786b817c8}{paint()}} method will automatically be called sometime later, by the message thread, to paint any bits that need refreshing. In JUCE (and almost all modern UI frameworks), you never redraw something synchronously.

You should never need to call this method directly -\/ to take a snapshot of the component you could use \mbox{\hyperlink{class_component_a275e46e5a8147f89c142c91ba5c793a5}{create\+Component\+Snapshot()}} or \mbox{\hyperlink{class_component_a2c960bd5ec5ac76d18e82b6883096714}{paint\+Entire\+Component()}}.
