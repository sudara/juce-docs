\doxysubsection{Detailed Description}
Represents a midi output device.

To create one of these, use the static \mbox{\hyperlink{class_midi_output_a7a4aa7efd875d463393e8946f0681506}{get\+Available\+Devices()}} method to find out what outputs are available, and then use the \mbox{\hyperlink{class_midi_output_ad2e44cf98e40d137bd43a063f785fd77}{open\+Device()}} method to try to open one.

{\footnotesize\ttfamily \mbox{\hyperlink{png_8h_aa8c59027f9ab2769342f248709d68d17}{void}} Midi\+Output\+::send\+Block\+Of\+Messages (\begin{DoxyParamCaption}\item[{\mbox{\hyperlink{zconf_8in_8h_a2c212835823e3c54a8ab6d95c652660e}{const}} \mbox{\hyperlink{class_midi_buffer}{Midi\+Buffer}} \&}]{buffer,  }\item[{double}]{millisecond\+Counter\+To\+Start\+At,  }\item[{double}]{samples\+Per\+Second\+For\+Buffer }\end{DoxyParamCaption})}

This lets you supply a block of messages that will be sent out at some point in the future.

The \mbox{\hyperlink{class_midi_output}{Midi\+Output}} class has an internal thread that can send out timestamped messages -\/ this appends a set of messages to its internal buffer, ready for sending.

This will only work if you\textquotesingle{}ve already started the thread with \mbox{\hyperlink{class_midi_output_ab4037a74a2ba5206ea12bc706132e9c6}{start\+Background\+Thread()}}.

A time is specified, at which the block of messages should be sent. This time uses the same time base as \mbox{\hyperlink{class_time_a66c3f2ac6b0a2693a8fc9506da584d27}{Time\+::get\+Millisecond\+Counter()}}, and must be in the future.

The samples\+Per\+Second\+For\+Buffer parameter indicates the number of samples per second used by the \mbox{\hyperlink{class_midi_buffer}{Midi\+Buffer}}. Each event in a \mbox{\hyperlink{class_midi_buffer}{Midi\+Buffer}} has a sample position, and the samples\+Per\+Second\+For\+Buffer value is needed to convert this sample position to a real time. \mbox{\Hypertarget{class_midi_output_ae54072205364d56260eaa11c7771289f}\label{class_midi_output_ae54072205364d56260eaa11c7771289f}}
