\doxysubsection{Detailed Description}
A base class for writing audio apps that stream from the audio i/o devices. Conveniently combines a \mbox{\hyperlink{class_component}{Component}} with an \mbox{\hyperlink{class_audio_source}{Audio\+Source}} to provide a starting point for your audio applications.

A subclass can inherit from this and implement just a few methods such as \mbox{\hyperlink{class_audio_app_component_aec5e36e694a5bd2edae903d1079166b1}{get\+Next\+Audio\+Block()}}. The base class provides a basic \mbox{\hyperlink{class_audio_device_manager}{Audio\+Device\+Manager}} object and runs audio through the default output device.

An application should only create one global instance of this object and multiple classes should not inherit from this.

This class should not be inherited when creating a plug-\/in as the host will handle audio streams from hardware devices.

Called repeatedly to fetch subsequent blocks of audio data.

After calling the \mbox{\hyperlink{class_audio_app_component_a70634aa3ffaf6e7ff0d233e5933a063d}{prepare\+To\+Play()}} method, this callback will be made each time the audio playback hardware (or whatever other destination the audio data is going to) needs another block of data.

It will generally be called on a high-\/priority system thread, or possibly even an interrupt, so be careful not to do too much work here, as that will cause audio glitches!

Tells the source to prepare for playing.

An \mbox{\hyperlink{class_audio_source}{Audio\+Source}} has two states\+: prepared and unprepared.

The \mbox{\hyperlink{class_audio_app_component_a70634aa3ffaf6e7ff0d233e5933a063d}{prepare\+To\+Play()}} method is guaranteed to be called at least once on an \textquotesingle{}unprepared\textquotesingle{} source to put it into a \textquotesingle{}prepared\textquotesingle{} state before any calls will be made to \mbox{\hyperlink{class_audio_app_component_aec5e36e694a5bd2edae903d1079166b1}{get\+Next\+Audio\+Block()}}. This callback allows the source to initialise any resources it might need when playing.

Once playback has finished, the \mbox{\hyperlink{class_audio_app_component_a588d9b990fc308774215548959e0fc50}{release\+Resources()}} method is called to put the stream back into an \textquotesingle{}unprepared\textquotesingle{} state.

Note that this method could be called more than once in succession without a matching call to \mbox{\hyperlink{class_audio_app_component_a588d9b990fc308774215548959e0fc50}{release\+Resources()}}, so make sure your code is robust and can handle that kind of situation.


\begin{DoxyParams}{Parameters}

This number may vary slightly, because it will be dependent on audio hardware callbacks, and these aren\textquotesingle{}t guaranteed to always use a constant block size, so the source should be able to cope with small variations. \\

Allows the source to release anything it no longer needs after playback has stopped.

This will be called when the source is no longer going to have its \mbox{\hyperlink{class_audio_app_component_aec5e36e694a5bd2edae903d1079166b1}{get\+Next\+Audio\+Block()}} method called, so it should release any spare memory, etc. that it might have allocated during the \mbox{\hyperlink{class_audio_app_component_a70634aa3ffaf6e7ff0d233e5933a063d}{prepare\+To\+Play()}} call.

Note that there\textquotesingle{}s no guarantee that \mbox{\hyperlink{class_audio_app_component_a70634aa3ffaf6e7ff0d233e5933a063d}{prepare\+To\+Play()}} will actually have been called before \mbox{\hyperlink{class_audio_app_component_a588d9b990fc308774215548959e0fc50}{release\+Resources()}}, and it may be called more than once in succession, so make sure your code is robust and doesn\textquotesingle{}t make any assumptions about when it will be called.
