class Reference\+Counted\+Array$<$ Object\+Class, Type\+Of\+Critical\+Section\+To\+Use $>$}
Holds a list of objects derived from \mbox{\hyperlink{class_reference_counted_object}{Reference\+Counted\+Object}}, or which implement basic reference-\/count handling methods.

The template parameter specifies the class of the object you want to point to -\/ the easiest way to make a class reference-\/countable is to simply make it inherit from \mbox{\hyperlink{class_reference_counted_object}{Reference\+Counted\+Object}} or \mbox{\hyperlink{class_single_threaded_reference_counted_object}{Single\+Threaded\+Reference\+Counted\+Object}}, but if you need to, you can roll your own reference-\/countable class by implementing a set of methods called inc\+Reference\+Count(), dec\+Reference\+Count(), and dec\+Reference\+Count\+Without\+Deleting(). See \mbox{\hyperlink{class_reference_counted_object}{Reference\+Counted\+Object}} for examples of how these methods should behave.

A \mbox{\hyperlink{class_reference_counted_array}{Reference\+Counted\+Array}} holds objects derived from \mbox{\hyperlink{class_reference_counted_object}{Reference\+Counted\+Object}}, and takes care of incrementing and decrementing their ref counts when they are added and removed from the array.

To make all the array\textquotesingle{}s methods thread-\/safe, pass in \char`\"{}\+Critical\+Section\char`\"{} as the templated Type\+Of\+Critical\+Section\+To\+Use parameter, instead of the default \mbox{\hyperlink{class_dummy_critical_section}{Dummy\+Critical\+Section}}.
