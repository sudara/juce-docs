\subsubsection*{template$<$class Element\+Type, class Type\+Of\+Critical\+Section\+To\+Use = Dummy\+Critical\+Section$>$\newline
class Sorted\+Set$<$ Element\+Type, Type\+Of\+Critical\+Section\+To\+Use $>$}
Holds a set of unique primitive objects, such as ints or doubles.

A set can only hold one item with a given value, so if for example it\textquotesingle{}s a set of integers, attempting to add the same integer twice will do nothing the second time.

Internally, the list of items is kept sorted (which means that whatever kind of primitive type is used must support the ==, \texorpdfstring{$<$}{<}, \texorpdfstring{$>$}{>}, \texorpdfstring{$<$}{<}= and \texorpdfstring{$>$}{>}= operators to determine the order), and searching the set for known values is very fast because it uses a binary-\/chop method.

Note that if you\textquotesingle{}re using a class or struct as the element type, it must be capable of being copied or moved with a straightforward memcpy, rather than needing construction and destruction code.

To make all the set\textquotesingle{}s methods thread-\/safe, pass in \char`\"{}\+Critical\+Section\char`\"{} as the templated Type\+Of\+Critical\+Section\+To\+Use parameter, instead of the default \mbox{\hyperlink{class_dummy_critical_section}{Dummy\+Critical\+Section}}.
