\doxysubsection{Detailed Description}
For laying out a set of components, where the components have preferred sizes and size limits, but where they are allowed to stretch to fill the available space.

For example, if you have a component containing several other components, and each one should be given a share of the total size, you could use one of these to resize the child components when the parent component is resized. Then you could add a \mbox{\hyperlink{class_stretchable_layout_resizer_bar}{Stretchable\+Layout\+Resizer\+Bar}} to easily let the user rescale them.

A \mbox{\hyperlink{class_stretchable_layout_manager}{Stretchable\+Layout\+Manager}} operates only in one dimension, so if you have a set of components stacked vertically on top of each other, you\textquotesingle{}d use one to manage their heights. To build up complex arrangements of components, e.\+g. for applications with multiple nested panels, you would use more than one \mbox{\hyperlink{class_stretchable_layout_manager}{Stretchable\+Layout\+Manager}}. E.\+g. by using two (one vertical, one horizontal), you could create a resizable spreadsheet-\/style table.
