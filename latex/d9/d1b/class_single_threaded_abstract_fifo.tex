\doxysubsection{Detailed Description}
Encapsulates the logic for a single-\/threaded FIFO.

This might be useful for building buffers which can be written and read in blocks of different sizes. For example, in an audio effect we might wish to run some processing on fixed-\/size blocks of audio input, but the host may provide input blocks of varying sizes. In this situation, we might want to store the previous input in a buffer, and extract a fixed-\/size block whenever there are enough samples available. The \mbox{\hyperlink{class_single_threaded_abstract_fifo}{Single\+Threaded\+Abstract\+Fifo}} implements logic suitable for this use-\/case.

This class is quite similar to \mbox{\hyperlink{class_abstract_fifo}{Abstract\+Fifo}}, in that it only keeps track of the current read/write locations. The user is responsible for providing the actual buffer that will be read/written.

The intended usage of this class is as follows\+:
\begin{DoxyItemize}
\item Create some backing storage in a vector, \mbox{\hyperlink{class_audio_buffer}{Audio\+Buffer}} etc.
\item Construct a \mbox{\hyperlink{class_single_threaded_abstract_fifo}{Single\+Threaded\+Abstract\+Fifo}} to manage the buffer, passing the number of items in the buffer.
\item Each time new input is ready, call \mbox{\hyperlink{class_single_threaded_abstract_fifo_aafd143b76bb2c54539d1bb7b2b05c1a9}{write()}}, passing the number of items you wish to write into the buffer. This function returns a pair of ranges describing which indices in the backing storage should be written.
\item Call \mbox{\hyperlink{class_single_threaded_abstract_fifo_ab83c943b44c120be35bf12943063500e}{get\+Num\+Readable()}} to find out how many items are ready to read from the buffer.
\item If there are enough items ready to read, call \mbox{\hyperlink{class_single_threaded_abstract_fifo_a91ec1d5f8a4a505169e69580a80cbf74}{read()}}, passing the number of items you require. This function returns a pair of ranges describing which indices in the backing storage may be read.
\end{DoxyItemize}

Unlike \mbox{\hyperlink{class_abstract_fifo}{Abstract\+Fifo}}, the \mbox{\hyperlink{class_single_threaded_abstract_fifo}{Single\+Threaded\+Abstract\+Fifo}} is intended for use from a single thread. It is not safe to call any non-\/const member function of \mbox{\hyperlink{class_single_threaded_abstract_fifo}{Single\+Threaded\+Abstract\+Fifo}} concurrently with any other member function.
