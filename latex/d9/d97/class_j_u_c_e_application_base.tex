\doxysubsection{Detailed Description}
Abstract base class for application classes.

Note that in the juce\+\_\+gui\+\_\+basics module, there\textquotesingle{}s a utility class \mbox{\hyperlink{class_j_u_c_e_application}{JUCEApplication}} which derives from \mbox{\hyperlink{class_j_u_c_e_application_base}{JUCEApplication\+Base}}, and takes care of a few chores. Most of the time you\textquotesingle{}ll want to derive your class from \mbox{\hyperlink{class_j_u_c_e_application}{JUCEApplication}} rather than using \mbox{\hyperlink{class_j_u_c_e_application_base}{JUCEApplication\+Base}} directly, but if you\textquotesingle{}re not using the juce\+\_\+gui\+\_\+basics module then you might need to go straight to this base class.

Any application that wants to run an event loop must declare a subclass of \mbox{\hyperlink{class_j_u_c_e_application_base}{JUCEApplication\+Base}}, and implement its various pure virtual methods.

It then needs to use the START\+\_\+\+JUCE\+\_\+\+APPLICATION macro somewhere in a CPP file to declare an instance of this class and generate suitable platform-\/specific boilerplate code to launch the app.

This will be called once to let the application do whatever initialisation it needs, create its windows, etc.

After the method returns, the normal event-\/dispatch loop will be run, until the \mbox{\hyperlink{class_j_u_c_e_application_base_a542e212605f0aa693f687d8798998615}{quit()}} method is called, at which point the \mbox{\hyperlink{class_j_u_c_e_application_base_a318021f86bf379caeb4cb27c32f0f229}{shutdown()}} method will be called to let the application clear up anything it needs to delete.

Signals that the main message loop should stop and the application should terminate.

This isn\textquotesingle{}t synchronous, it just posts a quit message to the main queue, and when this message arrives, the message loop will stop, the \mbox{\hyperlink{class_j_u_c_e_application_base_a318021f86bf379caeb4cb27c32f0f229}{shutdown()}} method will be called, and the app will exit.

Note that this will cause an unconditional quit to happen, so if you need an extra level before this, e.\+g. to give the user the chance to save their work and maybe cancel the quit, you\textquotesingle{}ll need to handle this in the \mbox{\hyperlink{class_j_u_c_e_application_base_aa4ae0dcc3467f4927d8365b4814b1396}{system\+Requested\+Quit()}} method -\/ see that method\textquotesingle{}s help for more info.

Called when the operating system is trying to close the application.

The default implementation of this method is to call \mbox{\hyperlink{class_j_u_c_e_application_base_a542e212605f0aa693f687d8798998615}{quit()}}, but it may be overloaded to ignore the request or do some other special behaviour instead. For example, you might want to offer the user the chance to save their changes before quitting, and give them the chance to cancel.

If you want to send a quit signal to your app, this is the correct method to call, because it means that requests that come from the system get handled in the same way as those from your own application code. So e.\+g. you\textquotesingle{}d call this method from a \char`\"{}quit\char`\"{} item on a menu bar.
