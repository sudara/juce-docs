\doxysubsection{Detailed Description}
Loads and maintains a tree of Components from a \mbox{\hyperlink{class_value_tree}{Value\+Tree}} that represents them.

To allow the state of a tree of components to be saved as a \mbox{\hyperlink{class_value_tree}{Value\+Tree}} and re-\/loaded, this class lets you register a set of type-\/handlers for the different components that are involved, and then uses these types to re-\/create a set of components from its stored state.

Essentially, to use this, you need to create a \mbox{\hyperlink{class_component_builder}{Component\+Builder}} with your \mbox{\hyperlink{class_value_tree}{Value\+Tree}}, then use \mbox{\hyperlink{class_component_builder_a97053a0ddf882ea7c6178ad06985865c}{register\+Type\+Handler()}} to give it a set of type handlers that can cope with all the items in your tree. Then you can call get\+Component() to build the component. Once you\textquotesingle{}ve got the component you can either take it and delete the \mbox{\hyperlink{class_component_builder}{Component\+Builder}} object, or if you keep the \mbox{\hyperlink{class_component_builder}{Component\+Builder}} around, it\textquotesingle{}ll monitor any changes in the \mbox{\hyperlink{class_value_tree}{Value\+Tree}} and automatically update the component to reflect these changes.

{\footnotesize\ttfamily \mbox{\hyperlink{class_component}{Component}} $\ast$ Component\+Builder\+::get\+Managed\+Component (\begin{DoxyParamCaption}{ }\end{DoxyParamCaption})}

Returns the builder\textquotesingle{}s component (creating it if necessary).

The first time that this method is called, the builder will attempt to create a component from the \mbox{\hyperlink{class_value_tree}{Value\+Tree}}, so you must have registered some suitable type handlers before calling this. If there\textquotesingle{}s a problem and the component can\textquotesingle{}t be created, this method returns nullptr.

The component that is returned is owned by this \mbox{\hyperlink{class_component_builder}{Component\+Builder}}, so you can put it inside your own parent components, but don\textquotesingle{}t delete it! The \mbox{\hyperlink{class_component_builder}{Component\+Builder}} will delete it automatically when the builder is destroyed. If you want to get a component that you can delete yourself, call \mbox{\hyperlink{class_component_builder_a31e05ef24135b58770e73d14748061c3}{create\+Component()}} instead.

The \mbox{\hyperlink{class_component_builder}{Component\+Builder}} will update this component if any changes are made to the \mbox{\hyperlink{class_value_tree}{Value\+Tree}}, so if there\textquotesingle{}s a chance that the tree might change, be careful not to keep any pointers to sub-\/components, as they may be changed or removed. \mbox{\Hypertarget{class_component_builder_a31c6a85cefbac6a0647c8c62d730bc90}\label{class_component_builder_a31c6a85cefbac6a0647c8c62d730bc90}}
