\doxysubsection{Detailed Description}
A window that displays a pop-\/up tooltip when the mouse hovers over another component.

To enable tooltips in your app, just create a single instance of a \mbox{\hyperlink{class_tooltip_window}{Tooltip\+Window}} object. Note that if you instantiate more than one instance of this class with the same parent\+Component (even if both \mbox{\hyperlink{class_tooltip_window}{Tooltip\+Window}}\textquotesingle{}s parent\+Component is nil), you\textquotesingle{}ll end up with multiple tooltips being shown! To avoid this use a \mbox{\hyperlink{class_shared_resource_pointer}{Shared\+Resource\+Pointer}} to instantiate the \mbox{\hyperlink{class_tooltip_window}{Tooltip\+Window}} only once.

For audio plug-\/ins (which should not be opening native windows) it is better to add a \mbox{\hyperlink{class_tooltip_window}{Tooltip\+Window}} as a member variable to the editor and ensure that the editor is the parent\+Component of your \mbox{\hyperlink{class_tooltip_window}{Tooltip\+Window}}. This will ensure that your \mbox{\hyperlink{class_tooltip_window}{Tooltip\+Window}} is scaled according to your editor and the DAWs scaling setting.

The \mbox{\hyperlink{class_tooltip_window}{Tooltip\+Window}} object will then stay invisible, waiting until the mouse hovers for the specified length of time -\/ it will then see if it\textquotesingle{}s currently over a component which implements the \mbox{\hyperlink{class_tooltip_client}{Tooltip\+Client}} interface, and if so, it will make itself visible to show the tooltip in the appropriate place.

Creates a tooltip window.

Make sure your app only creates one instance of this class, otherwise you\textquotesingle{}ll get multiple overlaid tooltips appearing. The window will initially be invisible and will make itself visible when it needs to display a tip.

To change the style of tooltips, see the \mbox{\hyperlink{class_look_and_feel}{Look\+And\+Feel}} class for its tooltip methods.
