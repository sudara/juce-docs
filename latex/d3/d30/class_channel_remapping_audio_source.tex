\doxysubsection{Detailed Description}
An \mbox{\hyperlink{class_audio_source}{Audio\+Source}} that takes the audio from another source, and re-\/maps its input and output channels to a different arrangement.

You can use this to increase or decrease the number of channels that an audio source uses, or to re-\/order those channels.

Call the \mbox{\hyperlink{namespacegl_a3b437ea454d772211296e220c025bdd2}{reset()}} method before using it to set up a default mapping, and then the \mbox{\hyperlink{class_channel_remapping_audio_source_a6d2601cc888b1c4d19ed4bcd4877cc53}{set\+Input\+Channel\+Mapping()}} and \mbox{\hyperlink{class_channel_remapping_audio_source_a319830bf0ca0a05bf60d070cf3d2ab3a}{set\+Output\+Channel\+Mapping()}} methods to create an appropriate mapping, otherwise no channels will be connected and it\textquotesingle{}ll produce silence.

Called repeatedly to fetch subsequent blocks of audio data.

After calling the \mbox{\hyperlink{class_channel_remapping_audio_source_abe7b07c27ff2987fd6968f1db40caafa}{prepare\+To\+Play()}} method, this callback will be made each time the audio playback hardware (or whatever other destination the audio data is going to) needs another block of data.

It will generally be called on a high-\/priority system thread, or possibly even an interrupt, so be careful not to do too much work here, as that will cause audio glitches!

Tells the source to prepare for playing.

An \mbox{\hyperlink{class_audio_source}{Audio\+Source}} has two states\+: prepared and unprepared.

The \mbox{\hyperlink{class_channel_remapping_audio_source_abe7b07c27ff2987fd6968f1db40caafa}{prepare\+To\+Play()}} method is guaranteed to be called at least once on an \textquotesingle{}unprepared\textquotesingle{} source to put it into a \textquotesingle{}prepared\textquotesingle{} state before any calls will be made to \mbox{\hyperlink{class_channel_remapping_audio_source_a7cc4e508799f79e4a7c750ea7a3c23f3}{get\+Next\+Audio\+Block()}}. This callback allows the source to initialise any resources it might need when playing.

Once playback has finished, the \mbox{\hyperlink{class_channel_remapping_audio_source_adec9a1e901cc8b88626f02aaa5582a85}{release\+Resources()}} method is called to put the stream back into an \textquotesingle{}unprepared\textquotesingle{} state.

Note that this method could be called more than once in succession without a matching call to \mbox{\hyperlink{class_channel_remapping_audio_source_adec9a1e901cc8b88626f02aaa5582a85}{release\+Resources()}}, so make sure your code is robust and can handle that kind of situation.

This number may vary slightly, because it will be dependent on audio hardware callbacks, and these aren\textquotesingle{}t guaranteed to always use a constant block size, so the source should be able to cope with small variations. \\
