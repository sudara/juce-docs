\doxysubsection{Detailed Description}
A resizable window with a title bar and maximise, minimise and close buttons.

This subclass of \mbox{\hyperlink{class_resizable_window}{Resizable\+Window}} creates a fairly standard type of window with a title bar and various buttons. The name of the component is shown in the title bar, and an icon can optionally be specified with \mbox{\hyperlink{class_document_window_ae2270940d36586af89359045bdd9c844}{set\+Icon()}}.

All the methods available to a \mbox{\hyperlink{class_resizable_window}{Resizable\+Window}} are also available to this, so it can easily be made resizable, minimised, maximised, etc.

It\textquotesingle{}s not advisable to add child components directly to a \mbox{\hyperlink{class_document_window}{Document\+Window}}\+: put them inside your content component instead. And overriding methods like \mbox{\hyperlink{class_document_window_a22979dafffb918b48641a7a90c14239b}{resized()}}, \mbox{\hyperlink{class_resizable_window_acfb2ced15e476e78f9a71b0214414798}{moved()}}, etc is also not recommended -\/ instead override these methods for your content component. (If for some obscure reason you do need to override these methods, always remember to call the super-\/class\textquotesingle{}s \mbox{\hyperlink{class_document_window_a22979dafffb918b48641a7a90c14239b}{resized()}} method too, otherwise it\textquotesingle{}ll fail to lay out the window decorations correctly).

You can also automatically add a menu bar to the window, using the \mbox{\hyperlink{class_document_window_a4a808defbae48839786354a350eaf7ea}{set\+Menu\+Bar()}} method.

{\footnotesize\ttfamily virtual \mbox{\hyperlink{png_8h_aa8c59027f9ab2769342f248709d68d17}{void}} Document\+Window\+::close\+Button\+Pressed (\begin{DoxyParamCaption}{ }\end{DoxyParamCaption})\hspace{0.3cm}{\ttfamily [virtual]}}

This method is called when the user tries to close the window.

This is triggered by the user clicking the close button, or using some other OS-\/specific key shortcut or OS menu for getting rid of a window.

If the window is just a pop-\/up, you should override this \mbox{\hyperlink{class_document_window_a71c46283a6fe63f2cad49d01997e16d5}{close\+Button\+Pressed()}} method and make it delete the window in whatever way is appropriate for your app. E.\+g. you might just want to call \char`\"{}delete this\char`\"{}.

If your app is centred around this window such that the whole app should quit when the window is closed, then you will probably want to use this method as an opportunity to call \mbox{\hyperlink{class_j_u_c_e_application_base_a542e212605f0aa693f687d8798998615}{JUCEApplication\+Base\+::quit()}}, and leave the window to be deleted later by your \mbox{\hyperlink{class_j_u_c_e_application_base_a318021f86bf379caeb4cb27c32f0f229}{JUCEApplication\+Base\+::shutdown()}} method. (Doing it this way means that your window will still get cleaned-\/up if the app is quit by some other means (e.\+g. a cmd-\/Q on the mac or closing it via the taskbar icon on Windows).

(Note that the \mbox{\hyperlink{class_document_window}{Document\+Window}} class overrides \mbox{\hyperlink{class_component_a543e70d5a3ff75cc3facb476f4cdb131}{Component\+::user\+Tried\+To\+Close\+Window()}} and redirects it to call this method, so any methods of closing the window that are caught by \mbox{\hyperlink{class_document_window_a056defe79da104beb80243a1416c1f8b}{user\+Tried\+To\+Close\+Window()}} will also end up here).
