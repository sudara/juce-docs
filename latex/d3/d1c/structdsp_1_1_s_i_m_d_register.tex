\doxysubsection{Detailed Description}
\subsubsection*{template$<$typename Type$>$\newline
struct dsp\+::\+SIMDRegister$<$ Type $>$}
A wrapper around the platform\textquotesingle{}s native SIMD register type.

This class is only available on SIMD machines. Use JUCE\+\_\+\+USE\+\_\+\+SIMD to query if SIMD is available for your system.

SIMDRegister$<$\+Type$>$ is a templated class representing the native vectorized version of Floating\+Type. \mbox{\hyperlink{structdsp_1_1_s_i_m_d_register}{SIMDRegister}} supports all numerical primitive types and std\+:complex$<$float$>$ and std\+::complex$<$double$>$ supports and most operations of the corresponding primitive type. Additionally, \mbox{\hyperlink{structdsp_1_1_s_i_m_d_register}{SIMDRegister}} can be accessed like an array to extract the individual elements.

If you are using \mbox{\hyperlink{structdsp_1_1_s_i_m_d_register}{SIMDRegister}} as a pointer, then you must ensure that the memory is sufficiently aligned for SIMD vector operations. Failing to do so will result in crashes or very slow code. Use \mbox{\hyperlink{structdsp_1_1_s_i_m_d_register_a2615629aee806ef6b0990301544d2170}{SIMDRegister\+::is\+SIMDAligned}} to query if a pointer is sufficiently aligned for SIMD vector operations.

Note that using \mbox{\hyperlink{structdsp_1_1_s_i_m_d_register}{SIMDRegister}} without enabling optimizations will result in code with very poor performance.
