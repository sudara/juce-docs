When a component is modal, this callback allows it to choose which other components can still receive events.

When a modal component is active and the user clicks on a non-\/modal component, this method is called on the modal component, and if it returns true, the event is allowed to reach its target. If it returns false, the event is blocked and the \mbox{\hyperlink{classjuce_1_1_drag_and_drop_container_1_1_drag_image_component_ab2f663301edf5ad2f8ba4883f8a20604}{input\+Attempt\+When\+Modal()}} callback is made.

It called by the \mbox{\hyperlink{class_component_a70fc33489ec892d615452dfdbd4687db}{is\+Currently\+Blocked\+By\+Another\+Modal\+Component()}} method. The default implementation just returns false in all cases.

Called when a key is pressed.

When a key is pressed, the component that has the keyboard focus will have this method called. Remember that a component will only be given the focus if its \mbox{\hyperlink{class_component_a6a1f21a76b971d54870cb5c32c041055}{set\+Wants\+Keyboard\+Focus()}} method has been used to enable this.

If your implementation returns true, the event will be consumed and not passed on to any other listeners. If it returns false, the key will be passed to any Key\+Listeners that have been registered with this component. As soon as one of these returns true, the process will stop, but if they all return false, the event will be passed upwards to this component\textquotesingle{}s parent, and so on.

The default implementation of this method does nothing and returns false.

{\footnotesize\ttfamily \mbox{\hyperlink{png_8h_aa8c59027f9ab2769342f248709d68d17}{void}} juce\+::\+Drag\+And\+Drop\+Container\+::\+Drag\+Image\+Component\+::paint (\begin{DoxyParamCaption}\item[{\mbox{\hyperlink{class_graphics}{Graphics}} \&}]{g }\end{DoxyParamCaption})\hspace{0.3cm}{\ttfamily [inline]}, {\ttfamily [override]}, {\ttfamily [virtual]}}

Components can override this method to draw their content.

The \mbox{\hyperlink{classjuce_1_1_drag_and_drop_container_1_1_drag_image_component_a5efebd77065751cf2b395d3df4279b5b}{paint()}} method gets called when a region of a component needs redrawing, either because the component\textquotesingle{}s \mbox{\hyperlink{class_component_af3ab3eabec93dd60faf983370f3a1206}{repaint()}} method has been called, or because something has happened on the screen that means a section of a window needs to be redrawn.

Any child components will draw themselves over whatever this method draws. If you need to paint over the top of your child components, you can also implement the \mbox{\hyperlink{class_component_aa15dc77c06b5a0a4192090b4a323f79c}{paint\+Over\+Children()}} method to do this.

If you want to cause a component to redraw itself, this is done asynchronously -\/ calling the \mbox{\hyperlink{class_component_af3ab3eabec93dd60faf983370f3a1206}{repaint()}} method marks a region of the component as \char`\"{}dirty\char`\"{}, and the \mbox{\hyperlink{classjuce_1_1_drag_and_drop_container_1_1_drag_image_component_a5efebd77065751cf2b395d3df4279b5b}{paint()}} method will automatically be called sometime later, by the message thread, to paint any bits that need refreshing. In JUCE (and almost all modern UI frameworks), you never redraw something synchronously.

You should never need to call this method directly -\/ to take a snapshot of the component you could use \mbox{\hyperlink{class_component_a275e46e5a8147f89c142c91ba5c793a5}{create\+Component\+Snapshot()}} or \mbox{\hyperlink{class_component_a2c960bd5ec5ac76d18e82b6883096714}{paint\+Entire\+Component()}}.
