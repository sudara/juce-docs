\DoxyCodeLine{14  \textcolor{comment}{/* png\_info is a structure that holds the information in a PNG file so}}
\DoxyCodeLine{15 \textcolor{comment}{ * that the application can find out the characteristics of the image.}}
\DoxyCodeLine{16 \textcolor{comment}{ * If you are reading the file, this structure will tell you what is}}
\DoxyCodeLine{17 \textcolor{comment}{ * in the PNG file.  If you are writing the file, fill in the information}}
\DoxyCodeLine{18 \textcolor{comment}{ * you want to put into the PNG file, using png\_set\_*() functions, then}}
\DoxyCodeLine{19 \textcolor{comment}{ * call png\_write\_info().}}
\DoxyCodeLine{20 \textcolor{comment}{ *}}
\DoxyCodeLine{21 \textcolor{comment}{ * The names chosen should be very close to the PNG specification, so}}
\DoxyCodeLine{22 \textcolor{comment}{ * consult that document for information about the meaning of each field.}}
\DoxyCodeLine{23 \textcolor{comment}{ *}}
\DoxyCodeLine{24 \textcolor{comment}{ * With libpng < 0.95, it was only possible to directly set and read the}}
\DoxyCodeLine{25 \textcolor{comment}{ * the values in the png\_info\_struct, which meant that the contents and}}
\DoxyCodeLine{26 \textcolor{comment}{ * order of the values had to remain fixed.  With libpng 0.95 and later,}}
\DoxyCodeLine{27 \textcolor{comment}{ * however, there are now functions that abstract the contents of}}
\DoxyCodeLine{28 \textcolor{comment}{ * png\_info\_struct from the application, so this makes it easier to use}}
\DoxyCodeLine{29 \textcolor{comment}{ * libpng with dynamic libraries, and even makes it possible to use}}
\DoxyCodeLine{30 \textcolor{comment}{ * libraries that don't have all of the libpng ancillary chunk-\/handing}}
\DoxyCodeLine{31 \textcolor{comment}{ * functionality.  In libpng-\/1.5.0 this was moved into a separate private}}
\DoxyCodeLine{32 \textcolor{comment}{ * file that is not visible to applications.}}
\DoxyCodeLine{33 \textcolor{comment}{ *}}
\DoxyCodeLine{34 \textcolor{comment}{ * The following members may have allocated storage attached that should be}}
\DoxyCodeLine{35 \textcolor{comment}{ * cleaned up before the structure is discarded: palette, trans, text,}}
\DoxyCodeLine{36 \textcolor{comment}{ * pcal\_purpose, pcal\_units, pcal\_params, hist, iccp\_name, iccp\_profile,}}
\DoxyCodeLine{37 \textcolor{comment}{ * splt\_palettes, scal\_unit, row\_pointers, and unknowns.   By default, these}}
\DoxyCodeLine{38 \textcolor{comment}{ * are automatically freed when the info structure is deallocated, if they were}}
\DoxyCodeLine{39 \textcolor{comment}{ * allocated internally by libpng.  This behavior can be changed by means}}
\DoxyCodeLine{40 \textcolor{comment}{ * of the png\_data\_freer() function.}}
\DoxyCodeLine{41 \textcolor{comment}{ *}}
\DoxyCodeLine{42 \textcolor{comment}{ * More allocation details: all the chunk-\/reading functions that}}
\DoxyCodeLine{43 \textcolor{comment}{ * change these members go through the corresponding png\_set\_*}}
\DoxyCodeLine{44 \textcolor{comment}{ * functions.  A function to clear these members is available: see}}
\DoxyCodeLine{45 \textcolor{comment}{ * png\_free\_data().  The png\_set\_* functions do not depend on being}}
\DoxyCodeLine{46 \textcolor{comment}{ * able to point info structure members to any of the storage they are}}
\DoxyCodeLine{47 \textcolor{comment}{ * passed (they make their own copies), EXCEPT that the png\_set\_text}}
\DoxyCodeLine{48 \textcolor{comment}{ * functions use the same storage passed to them in the text\_ptr or}}
\DoxyCodeLine{49 \textcolor{comment}{ * itxt\_ptr structure argument, and the png\_set\_rows and png\_set\_unknowns}}
\DoxyCodeLine{50 \textcolor{comment}{ * functions do not make their own copies.}}
\DoxyCodeLine{51 \textcolor{comment}{ */}}
