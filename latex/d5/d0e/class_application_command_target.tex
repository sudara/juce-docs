\doxysubsection{Detailed Description}
A command target publishes a list of command IDs that it can perform.

An \mbox{\hyperlink{class_application_command_manager}{Application\+Command\+Manager}} despatches commands to targets, which must be able to provide information about what commands they can handle.

To create a target, you\textquotesingle{}ll need to inherit from this class, implementing all of its pure virtual methods.

For info about how a target is chosen to receive a command, see \mbox{\hyperlink{class_application_command_manager_af8fb9732eaddb5f8df844f3915f0e914}{Application\+Command\+Manager\+::get\+First\+Command\+Target()}}.

This must provide details about one of the commands that this target can perform.

This will be called with one of the command IDs that the target provided in its \mbox{\hyperlink{class_application_command_target_a12b1c7a6abe2e5a5f27fda4a292fe33d}{get\+All\+Commands()}} methods.

It should fill-\/in all appropriate fields of the \mbox{\hyperlink{struct_application_command_info}{Application\+Command\+Info}} structure with suitable information about the command. (The command\+ID field will already have been filled-\/in by the caller).

The easiest way to set the info is using the \mbox{\hyperlink{struct_application_command_info_a3e9cd7e72fbf617a0d70d6be3d513b82}{Application\+Command\+Info\+::set\+Info()}} method to set all the fields at once.

If the command is currently inactive for some reason, this method must use \mbox{\hyperlink{struct_application_command_info_a68a043a95e058beeeddfba2441ee7306}{Application\+Command\+Info\+::set\+Active()}} to make that clear, (or it should set the is\+Disabled bit of the \mbox{\hyperlink{struct_application_command_info_a45fa351a09f7a6df3e76fe466c476905}{Application\+Command\+Info\+::flags}} field).

Any default key-\/presses for the command should be appended to the \mbox{\hyperlink{struct_application_command_info_acae1903bdfd8214162adb8326d1ee9da}{Application\+Command\+Info\+::default\+Keypresses}} field.

Note that if you change something that affects the status of the commands that would be returned by this method (e.\+g. something that makes some commands active or inactive), you should call \mbox{\hyperlink{class_application_command_manager_a91fef61705d98da881276733e304e7af}{Application\+Command\+Manager\+::command\+Status\+Changed()}} to cause the manager to refresh its status. 

Implemented in \mbox{\hyperlink{class_j_u_c_e_application_ae3222d0245de1d1baf346addf9d00f2e}{JUCEApplication}}, and \mbox{\hyperlink{class_code_editor_component_a9454244abc456d67d191c4f294858ad8}{Code\+Editor\+Component}}.

This must return the next target to try after this one.

When a command is being sent, and the first target can\textquotesingle{}t handle that command, this method is used to determine the next target that should be tried.

It may return nullptr if it doesn\textquotesingle{}t know of another target.

If your target is a \mbox{\hyperlink{class_component}{Component}}, you would usually use the \mbox{\hyperlink{class_application_command_target_ae22a2eb02affb6278074067f15abb58c}{find\+First\+Target\+Parent\+Component()}} method to return a parent component that might want to handle it.

Makes this target invoke a command.

Your code can call this method to invoke a command on this target, but normally you\textquotesingle{}d call it indirectly via \mbox{\hyperlink{class_application_command_manager_a6b506331c1cfb3d07b2925ac16b8141e}{Application\+Command\+Manager\+::invoke()}} or \mbox{\hyperlink{class_application_command_manager_a55f04432929174c2de6d622cb2d23c67}{Application\+Command\+Manager\+::invoke\+Directly()}}.

If this target can perform the given command, it will call its \mbox{\hyperlink{class_application_command_target_a0e8f962cda82e8aa1207d38361ff96f9}{perform()}} method to do so. If not, then \mbox{\hyperlink{class_application_command_target_a8def2ea10aad1a34d37f9afe1675b06d}{get\+Next\+Command\+Target()}} will be used to determine the next target to try, and the command will be passed along to it.
