\doxysubsection{Detailed Description}
This class is now deprecated in favour of \mbox{\hyperlink{class_ranged_directory_iterator}{Ranged\+Directory\+Iterator}}.

Searches through the files in a directory, returning each file that is found.

A \mbox{\hyperlink{class_directory_iterator}{Directory\+Iterator}} will search through a directory and its subdirectories using a wildcard filepattern match.

The iterator keeps track of directories that it has previously traversed, and will skip any previously-\/seen directories in the case of cycles caused by symbolic links. It is also possible to avoid following symbolic links altogether.

If you may be scanning a large number of files, it\textquotesingle{}s usually smarter to use this class than \mbox{\hyperlink{class_file_a5290046f7f78fffce0c4debd6668cbe5}{File\+::find\+Child\+Files()}} because it allows you to stop at any time, rather than having to wait for the entire scan to finish before getting the results.

Please note that the order in which files are returned is completely undefined! They\textquotesingle{}ll arrive in whatever order the underlying OS calls provide them, which will depend on the filesystem and other factors. If you need a sorted list, you\textquotesingle{}ll need to manually sort them using your preferred comparator after collecting the list.

It also provides an estimate of its progress, using a (highly inaccurate!) algorithm.

Moves the iterator along to the next file, and returns various properties of that file.

If you need to find out details about the file, it\textquotesingle{}s more efficient to call this method than to call the normal \mbox{\hyperlink{class_directory_iterator_a4d38dc126a3b4855dbbef96d8b31ae76}{next()}} method and then find out the details afterwards.

All the parameters are optional, so pass null pointers for any items that you\textquotesingle{}re not interested in.

\begin{DoxyReturn}{Returns}
true if a file was found (you can then use \mbox{\hyperlink{class_directory_iterator_af08ce7fe18fe425e84fb1b1f44fc8adb}{get\+File()}} to see what it was) -\/ or false if there are no more matching files. If it returns false, then none of the parameters will be filled-\/in. 
\end{DoxyReturn}
