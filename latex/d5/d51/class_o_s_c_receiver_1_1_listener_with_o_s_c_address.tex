\doxysubsection{Detailed Description}
\subsubsection*{template$<$typename Callback\+Type = Message\+Loop\+Callback$>$\newline
class OSCReceiver\+::\+Listener\+With\+OSCAddress$<$ Callback\+Type $>$}
A class for receiving only those OSC messages from an \mbox{\hyperlink{class_o_s_c_receiver}{OSCReceiver}} that match a given OSC address.

Use this class if your app receives OSC messages with different address patterns (for example \char`\"{}/juce/fader1\char`\"{}, /juce/knob2\char`\"{} etc.) and you want to route those to different objects. This class contains pre-\/build functionality for that OSC address routing, including wildcard pattern matching (e.\+g. \char`\"{}/juce/fader\mbox{[}0-\/9\mbox{]}\char`\"{}).  \+This class implements the concept of an \char`\"{}OSC Method"{} from the Open\+Sound\+Control 1.\+0 specification.

The template argument Callback\+Type determines how the callback will be called and has to be either \mbox{\hyperlink{struct_o_s_c_receiver_1_1_message_loop_callback}{Message\+Loop\+Callback}} or \mbox{\hyperlink{struct_o_s_c_receiver_1_1_realtime_callback}{Realtime\+Callback}}. If not specified, \mbox{\hyperlink{struct_o_s_c_receiver_1_1_message_loop_callback}{Message\+Loop\+Callback}} will be used by default.

Note\+: This type of listener will ignore OSC bundles.
