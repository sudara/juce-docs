\subsubsection*{template$<$class Listener\+Class, class Array\+Type = Array$<$\+Listener\+Class$\ast$$>$$>$\newline
class Listener\+List$<$ Listener\+Class, Array\+Type $>$}
Holds a set of objects and can invoke a member function callback on each object in the set with a single call.

Use a \mbox{\hyperlink{class_listener_list}{Listener\+List}} to manage a set of objects which need a callback, and you can invoke a member function by simply calling \mbox{\hyperlink{class_listener_list_adea25521b1707f2f5ae242b3353f5751}{call()}} or \mbox{\hyperlink{class_listener_list_ac89398e526d746817d7f2f488461bead}{call\+Checked()}}.

E.\+g. 
\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{keyword}{class }MyListenerType}
\DoxyCodeLine{\{}
\DoxyCodeLine{\textcolor{keyword}{public}:}
\DoxyCodeLine{    \textcolor{keywordtype}{void} myCallbackMethod (\textcolor{keywordtype}{int} foo, \textcolor{keywordtype}{bool} bar);}
\DoxyCodeLine{\};}
\DoxyCodeLine{}
\DoxyCodeLine{\mbox{\hyperlink{class_listener_list}{ListenerList<MyListenerType>}} listeners;}
\DoxyCodeLine{listeners.\mbox{\hyperlink{class_listener_list_a426564389022f07722ed9e94e07aabfe}{add}} (someCallbackObjects...);}
\DoxyCodeLine{}
\DoxyCodeLine{\textcolor{comment}{// This will invoke myCallbackMethod (1234, true) on each of the objects}}
\DoxyCodeLine{\textcolor{comment}{// in the list...}}
\DoxyCodeLine{listeners.\mbox{\hyperlink{class_listener_list_adea25521b1707f2f5ae242b3353f5751}{call}} ([] (MyListenerType\& l) \{ l.myCallbackMethod (1234, \textcolor{keyword}{true}); \});}

\end{DoxyCode}


If you add or remove listeners from the list during one of the callbacks -\/ i.\+e. while it\textquotesingle{}s in the middle of iterating the listeners, then it\textquotesingle{}s guaranteed that no listeners will be mistakenly called after they\textquotesingle{}ve been removed, but it may mean that some of the listeners could be called more than once, or not at all, depending on the list\textquotesingle{}s order.

Sometimes, there\textquotesingle{}s a chance that invoking one of the callbacks might result in the list itself being deleted while it\textquotesingle{}s still iterating -\/ to survive this situation, you can use \mbox{\hyperlink{class_listener_list_ac89398e526d746817d7f2f488461bead}{call\+Checked()}} instead of \mbox{\hyperlink{class_listener_list_adea25521b1707f2f5ae242b3353f5751}{call()}}, passing it a local object to act as a \char`\"{}\+Bail\+Out\+Checker\char`\"{}. The Bail\+Out\+Checker must implement a method of the form \char`\"{}bool should\+Bail\+Out()\char`\"{}, and the list will check this after each callback to determine whether it should abort the operation. For an example of a bail-\/out checker, see the \mbox{\hyperlink{class_component_1_1_bail_out_checker}{Component\+::\+Bail\+Out\+Checker}} class, which can be used to check when a \mbox{\hyperlink{class_component}{Component}} has been deleted. See also \mbox{\hyperlink{struct_listener_list_1_1_dummy_bail_out_checker}{Listener\+List\+::\+Dummy\+Bail\+Out\+Checker}}, which is a dummy checker that always returns false.
