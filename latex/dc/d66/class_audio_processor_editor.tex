\doxysubsection{Detailed Description}
Base class for the component that acts as the GUI for an \mbox{\hyperlink{class_audio_processor}{Audio\+Processor}}.

Derive your editor component from this class, and create an instance of it by overriding the \mbox{\hyperlink{class_audio_processor_adf76a112d670baa25b40d13cf87e1306}{Audio\+Processor\+::create\+Editor()}} method.

This sets the maximum and minimum sizes for the window.

If the window\textquotesingle{}s current size is outside these limits, it will be resized to make sure it\textquotesingle{}s within them.

If you pass in a different minimum and maximum size, this will mark the editor as resizable by the host.

A direct call to \mbox{\hyperlink{class_component_a64cbeee83a441e20ed5e7acb600ae107}{set\+Bounds()}} will bypass any constraint checks, but when the window is dragged by the user or resized by other indirect means, the constrainer will limit the numbers involved.

Note that if you have set a custom constrainer for this editor then this will have no effect, and if you have removed the constrainer with {\ttfamily set\+Constrainer (nullptr);} then this will re-\/add the default constrainer with the new limits.

Currently only AUv3 plug-\/ins compiled for Mac\+OS 10.\+13 or i\+OS 11.\+0 (or later) support this functionality, and even then the host may choose to ignore this information.

The default behaviour is to report support for both cases.
