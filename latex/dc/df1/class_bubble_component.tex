\doxysubsection{Detailed Description}
A component for showing a message or other graphics inside a speech-\/bubble-\/shaped outline, pointing at a location on the screen.

This is a base class that just draws and positions the bubble shape, but leaves the drawing of any content up to a subclass. See \mbox{\hyperlink{class_bubble_message_component}{Bubble\+Message\+Component}} for a subclass that draws a text message.

To use it, create your subclass, then either add it to a parent component or put it on the desktop with add\+To\+Desktop (0), use \mbox{\hyperlink{class_bubble_component_a279ce3e2971a1f48711260f513c5508f}{set\+Position()}} to resize and position it, then make it visible.

Components can override this method to draw their content.

The \mbox{\hyperlink{class_bubble_component_ab7f569418ed2401a94663658622703d5}{paint()}} method gets called when a region of a component needs redrawing, either because the component\textquotesingle{}s \mbox{\hyperlink{class_component_af3ab3eabec93dd60faf983370f3a1206}{repaint()}} method has been called, or because something has happened on the screen that means a section of a window needs to be redrawn.

Any child components will draw themselves over whatever this method draws. If you need to paint over the top of your child components, you can also implement the \mbox{\hyperlink{class_component_aa15dc77c06b5a0a4192090b4a323f79c}{paint\+Over\+Children()}} method to do this.

If you want to cause a component to redraw itself, this is done asynchronously -\/ calling the \mbox{\hyperlink{class_component_af3ab3eabec93dd60faf983370f3a1206}{repaint()}} method marks a region of the component as \char`\"{}dirty\char`\"{}, and the \mbox{\hyperlink{class_bubble_component_ab7f569418ed2401a94663658622703d5}{paint()}} method will automatically be called sometime later, by the message thread, to paint any bits that need refreshing. In JUCE (and almost all modern UI frameworks), you never redraw something synchronously.

You should never need to call this method directly -\/ to take a snapshot of the component you could use \mbox{\hyperlink{class_component_a275e46e5a8147f89c142c91ba5c793a5}{create\+Component\+Snapshot()}} or \mbox{\hyperlink{class_component_a2c960bd5ec5ac76d18e82b6883096714}{paint\+Entire\+Component()}}.

Tells the bubble which positions it\textquotesingle{}s allowed to put itself in, relative to the point at which it\textquotesingle{}s pointing.

By default when \mbox{\hyperlink{class_bubble_component_a279ce3e2971a1f48711260f513c5508f}{set\+Position()}} is called, the bubble will place itself either above, below, left, or right of the target area. You can pass in a bitwise-\/\textquotesingle{}or\textquotesingle{} of the values in Bubble\+Placement to restrict this choice.

Moves and resizes the bubble to point at a given rectangle.

This will resize the bubble to fit its content, then find a position for it so that it\textquotesingle{}s next to, but doesn\textquotesingle{}t overlap the given rectangle. The rectangle\textquotesingle{}s coordinates are relative to either the bubble component\textquotesingle{}s parent component if it has one, or they are screen coordinates if not.

It\textquotesingle{}ll put itself either above, below, or to the side of the component depending on where there\textquotesingle{}s the most space, honouring any restrictions that were set with \mbox{\hyperlink{class_bubble_component_a7c0081674149dc2a8380a0b20cf6f6ae}{set\+Allowed\+Placement()}}.

distance\+From\+Target is the amount of space to leave between the bubble and the target rectangle, and arrow\+Length is the length of the arrow that it will draw.
