\doxysubsection{Detailed Description}
Manages a simple two-\/way messaging connection to another process, using either a socket or a named pipe as the transport medium.

To connect to a waiting socket or an open pipe, use the \mbox{\hyperlink{class_interprocess_connection_a007450a6ca6dc60bb48d384479d9a49d}{connect\+To\+Socket()}} or \mbox{\hyperlink{class_interprocess_connection_abafb2228ed136a2b34e2de03709eb9a7}{connect\+To\+Pipe()}} methods. If this succeeds, messages can be sent to the other end, and incoming messages will result in a callback via the \mbox{\hyperlink{class_interprocess_connection_a69549dd1bda0584d212940d3faf7457a}{message\+Received()}} method.

To open a pipe and wait for another client to connect to it, use the \mbox{\hyperlink{class_interprocess_connection_a0e073a6609a68de2e4fcc892c1d003c0}{create\+Pipe()}} method.

To act as a socket server and create connections for one or more client, see the \mbox{\hyperlink{class_interprocess_connection_server}{Interprocess\+Connection\+Server}} class.

IMPORTANT NOTE\+: Your derived Connection class {\itshape must} call {\ttfamily disconnect} in its destructor in order to cancel any pending messages before the class is destroyed.

When the object at the other end of this connection sends us a message with \mbox{\hyperlink{class_interprocess_connection_a6fc9e7b55c25eac48baebd131ded9d4f}{send\+Message()}}, this callback is used to deliver it to us.

If the connection was created with the callbacks\+On\+Message\+Thread flag set, then this will be called on the message thread; otherwise it will be called on a server thread.
