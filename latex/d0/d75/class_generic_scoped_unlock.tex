class Generic\+Scoped\+Unlock$<$ Lock\+Type $>$}
Automatically unlocks and re-\/locks a mutex object.

This is the reverse of a \mbox{\hyperlink{class_generic_scoped_lock}{Generic\+Scoped\+Lock}} object -\/ instead of locking the mutex for the lifetime of this object, it unlocks it.

Make sure you don\textquotesingle{}t try to unlock mutexes that aren\textquotesingle{}t actually locked!

Creates a \mbox{\hyperlink{class_generic_scoped_unlock}{Generic\+Scoped\+Unlock}}.

As soon as it is created, this will unlock the \mbox{\hyperlink{class_critical_section}{Critical\+Section}}, and when the Scoped\+Lock object is deleted, the \mbox{\hyperlink{class_critical_section}{Critical\+Section}} will be re-\/locked.

Make sure this object is created and deleted by the same thread, otherwise there are no guarantees what will happen! Best just to use it as a local stack object, rather than creating one with the new() operator. \mbox{\Hypertarget{class_generic_scoped_unlock_ab979a8270fadb8bd835ef693cb5ad474}\label{class_generic_scoped_unlock_ab979a8270fadb8bd835ef693cb5ad474}}
