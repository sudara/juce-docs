Extended host callback interface for an edit controller\+: \mbox{\hyperlink{class_steinberg_1_1_vst_1_1_i_component_handler2}{Vst\+::\+IComponent\+Handler2}}


\begin{DoxyItemize}

One part handles\+:
\begin{DoxyItemize}
\item Setting dirty state of the plug-\/in
\item Requesting the host to open the editor
\end{DoxyItemize}

The other part handles parameter group editing from the plug-\/in UI. It wraps a set of \mbox{\hyperlink{class_steinberg_1_1_vst_1_1_i_component_handler_adb54ddee9c7937386de45bc15f98e7da}{IComponent\+Handler\+::begin\+Edit}} / \mbox{\hyperlink{class_steinberg_1_1_vst_1_1_i_component_handler_afe59eb77360c51a5b4bea157a41bb107}{Steinberg\+::\+Vst\+::\+IComponent\+Handler\+::perform\+Edit}} / \mbox{\hyperlink{class_steinberg_1_1_vst_1_1_i_component_handler_ae50c7fa7245712f3134ed4506e9f434f}{Steinberg\+::\+Vst\+::\+IComponent\+Handler\+::end\+Edit}} functions (see \mbox{\hyperlink{class_steinberg_1_1_vst_1_1_i_component_handler}{IComponent\+Handler}}) which should use the same timestamp in the host when writing automation.
