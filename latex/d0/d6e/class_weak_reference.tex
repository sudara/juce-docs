\doxysubsection{Detailed Description}
\subsubsection*{template$<$class Object\+Type, class Reference\+Counting\+Type = Reference\+Counted\+Object$>$\newline
class Weak\+Reference$<$ Object\+Type, Reference\+Counting\+Type $>$}
This class acts as a pointer which will automatically become null if the object to which it points is deleted.

To accomplish this, the source object needs to cooperate by performing a couple of simple tasks. It must embed a \mbox{\hyperlink{class_weak_reference_1_1_master}{Weak\+Reference\+::\+Master}} object, which stores a shared pointer object, and must clear this master pointer in its destructor.

Note that \mbox{\hyperlink{class_weak_reference}{Weak\+Reference}} is not designed to be thread-\/safe, so if you\textquotesingle{}re accessing it from different threads, you\textquotesingle{}ll need to do your own locking around all uses of the pointer and the object it refers to.

This returns true if this reference has been pointing at an object, but that object has since been deleted.

If this reference was only ever pointing at a null pointer, this will return false. Using \mbox{\hyperlink{class_weak_reference_af85cf5fb4349bc5d186e3586fbe4be67}{operator=()}} to make this refer to a different object will reset this flag to match the status of the reference from which you\textquotesingle{}re copying.
