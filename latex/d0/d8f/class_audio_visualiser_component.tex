\doxysubsection{Detailed Description}
A simple component that can be used to show a scrolling waveform of audio data.

This is a handy way to get a quick visualisation of some audio data. Just create one of these, set its size and oversampling rate, and then feed it with incoming data by calling one of its \mbox{\hyperlink{class_audio_visualiser_component_a0667af6eb65c167a95a6a7b51b8339e0}{push\+Buffer()}} or \mbox{\hyperlink{class_audio_visualiser_component_aa0e53a6298d7cd56112fa004a933e598}{push\+Sample()}} methods.

You can override its paint method for more customised views, but it\textquotesingle{}s only designed as a quick-\/and-\/dirty class for simple tasks, so please don\textquotesingle{}t send us feature requests for fancy additional features that you\textquotesingle{}d like it to support! If you\textquotesingle{}re building a real-\/world app that requires more powerful waveform display, you\textquotesingle{}ll probably want to create your own component instead.

Components can override this method to draw their content.

The \mbox{\hyperlink{class_audio_visualiser_component_a4c9e47c97200c8bfcd43c47a71e4121d}{paint()}} method gets called when a region of a component needs redrawing, either because the component\textquotesingle{}s \mbox{\hyperlink{class_component_af3ab3eabec93dd60faf983370f3a1206}{repaint()}} method has been called, or because something has happened on the screen that means a section of a window needs to be redrawn.

Any child components will draw themselves over whatever this method draws. If you need to paint over the top of your child components, you can also implement the \mbox{\hyperlink{class_component_aa15dc77c06b5a0a4192090b4a323f79c}{paint\+Over\+Children()}} method to do this.

If you want to cause a component to redraw itself, this is done asynchronously -\/ calling the \mbox{\hyperlink{class_component_af3ab3eabec93dd60faf983370f3a1206}{repaint()}} method marks a region of the component as \char`\"{}dirty\char`\"{}, and the \mbox{\hyperlink{class_audio_visualiser_component_a4c9e47c97200c8bfcd43c47a71e4121d}{paint()}} method will automatically be called sometime later, by the message thread, to paint any bits that need refreshing. In JUCE (and almost all modern UI frameworks), you never redraw something synchronously.

You should never need to call this method directly -\/ to take a snapshot of the component you could use \mbox{\hyperlink{class_component_a275e46e5a8147f89c142c91ba5c793a5}{create\+Component\+Snapshot()}} or \mbox{\hyperlink{class_component_a2c960bd5ec5ac76d18e82b6883096714}{paint\+Entire\+Component()}}.
