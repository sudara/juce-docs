\doxysubsection{Detailed Description}
Represents a shared variant value.

A \mbox{\hyperlink{class_value}{Value}} object contains a reference to a var object, and can get and set its value. Listeners can be attached to be told when the value is changed.

The \mbox{\hyperlink{class_value}{Value}} class is a wrapper around a shared, reference-\/counted underlying data object -\/ this means that multiple \mbox{\hyperlink{class_value}{Value}} objects can all refer to the same piece of data, allowing all of them to be notified when any of them changes it.

When you create a \mbox{\hyperlink{class_value}{Value}} with its default constructor, it acts as a wrapper around a simple var object, but by creating a \mbox{\hyperlink{class_value}{Value}} that refers to a custom subclass of \mbox{\hyperlink{class_value_1_1_value_source}{Value\+Source}}, you can map the \mbox{\hyperlink{class_value}{Value}} onto any kind of underlying data.

Important note! The \mbox{\hyperlink{class_value}{Value}} class is not thread-\/safe! If you\textquotesingle{}re accessing one from multiple threads, then you\textquotesingle{}ll need to use your own synchronisation around any code that accesses it.

Adds a listener to receive callbacks when the value changes.

The listener is added to this specific \mbox{\hyperlink{class_value}{Value}} object, and not to the shared object that it refers to. When this object is deleted, all the listeners will be lost, even if other references to the same \mbox{\hyperlink{class_value}{Value}} still exist. So when you\textquotesingle{}re adding a listener, make sure that you add it to a \mbox{\hyperlink{class_value}{Value}} instance that will last for as long as you need the listener. In general, you\textquotesingle{}d never want to add a listener to a local stack-\/based \mbox{\hyperlink{class_value}{Value}}, but more likely to one that\textquotesingle{}s a member variable.
