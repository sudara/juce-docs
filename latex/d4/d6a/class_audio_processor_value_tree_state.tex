\doxysubsection{Detailed Description}
This class contains a \mbox{\hyperlink{class_value_tree}{Value\+Tree}} that is used to manage an \mbox{\hyperlink{class_audio_processor}{Audio\+Processor}}\textquotesingle{}s entire state.

It has its own internal class of parameter object that is linked to values within its \mbox{\hyperlink{class_value_tree}{Value\+Tree}}, and which are each identified by a string ID.

You can get access to the underlying \mbox{\hyperlink{class_value_tree}{Value\+Tree}} object via the state member variable, so you can add extra properties to it as necessary.

It also provides some utility child classes for connecting parameters directly to GUI controls like sliders.

The favoured constructor of this class takes a collection of Ranged\+Audio\+Parameters or Audio\+Processor\+Parameter\+Groups of Ranged\+Audio\+Parameters and adds them to the attached \mbox{\hyperlink{class_audio_processor}{Audio\+Processor}} directly.

The deprecated way of using this class is as follows\+:

1) Create an \mbox{\hyperlink{class_audio_processor_value_tree_state}{Audio\+Processor\+Value\+Tree\+State}}, and give it some parameters using \mbox{\hyperlink{class_audio_processor_value_tree_state_af43f4eabc224eddfe8a7fb39e23681f7}{create\+And\+Add\+Parameter()}}. 2) Initialise the state member variable with a type name.

The deprecated constructor will be removed from the API in a future version of JUCE!

Creates a state object for a given processor, and sets up all the parameters that will control that processor.

You should {\itshape not} assign a new \mbox{\hyperlink{class_value_tree}{Value\+Tree}} to the state, or call create\+And\+Add\+Parameter, after using this constructor.

Note that each \mbox{\hyperlink{class_audio_processor_value_tree_state}{Audio\+Processor\+Value\+Tree\+State}} should be attached to only one processor, and must have the same lifetime as the processor, as they will have dependencies on each other.

Returns a copy of the state value tree.

The \mbox{\hyperlink{class_audio_processor_value_tree_state}{Audio\+Processor\+Value\+Tree\+State}}\textquotesingle{}s \mbox{\hyperlink{class_value_tree}{Value\+Tree}} is updated internally on the message thread, but there may be cases when you may want to access the state from a different thread (get\+State\+Information is a good example). This method flushes all pending audio parameter value updates and returns a copy of the state in a thread safe way.

Note\+: This method uses locks to synchronise thread access, so whilst it is thread-\/safe, it is not realtime-\/safe. Replaces the state value tree.

The \mbox{\hyperlink{class_audio_processor_value_tree_state}{Audio\+Processor\+Value\+Tree\+State}}\textquotesingle{}s \mbox{\hyperlink{class_value_tree}{Value\+Tree}} is updated internally on the message thread, but there may be cases when you may want to modify the state from a different thread (set\+State\+Information is a good example). This method allows you to replace the state in a thread safe way.

Note\+: This method uses locks to synchronise thread access, so whilst it is thread-\/safe, it is not realtime-\/safe. Do not call this method from within your audio processing code!
