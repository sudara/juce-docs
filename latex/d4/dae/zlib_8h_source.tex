\DoxyCodeLine{44 \textcolor{comment}{/*}}
\DoxyCodeLine{45 \textcolor{comment}{     The 'zlib' compression library provides in-\/memory compression and}}
\DoxyCodeLine{46 \textcolor{comment}{  decompression functions, including integrity checks of the uncompressed}}
\DoxyCodeLine{47 \textcolor{comment}{  data.  This version of the library supports only one compression method}}
\DoxyCodeLine{48 \textcolor{comment}{  (deflation) but other algorithms will be added later and will have the same}}
\DoxyCodeLine{49 \textcolor{comment}{  stream interface.}}
\DoxyCodeLine{50 \textcolor{comment}{}}
\DoxyCodeLine{51 \textcolor{comment}{     Compression can be done in a single step if the buffers are large}}
\DoxyCodeLine{52 \textcolor{comment}{  enough (for example if an input file is mmap'ed), or can be done by}}
\DoxyCodeLine{53 \textcolor{comment}{  repeated calls of the compression function.  In the latter case, the}}
\DoxyCodeLine{54 \textcolor{comment}{  application must provide more input and/or consume the output}}
\DoxyCodeLine{55 \textcolor{comment}{  (providing more output space) before each call.}}
\DoxyCodeLine{56 \textcolor{comment}{}}
\DoxyCodeLine{57 \textcolor{comment}{     The compressed data format used by default by the in-\/memory functions is}}
\DoxyCodeLine{58 \textcolor{comment}{  the zlib format, which is a zlib wrapper documented in RFC 1950, wrapped}}
\DoxyCodeLine{59 \textcolor{comment}{  around a deflate stream, which is itself documented in RFC 1951.}}
\DoxyCodeLine{60 \textcolor{comment}{}}
\DoxyCodeLine{61 \textcolor{comment}{     The library also supports reading and writing files in gzip (.gz) format}}
\DoxyCodeLine{62 \textcolor{comment}{  with an interface similar to that of stdio using the functions that start}}
\DoxyCodeLine{63 \textcolor{comment}{  with "{}gz"{}.  The gzip format is different from the zlib format.  gzip is a}}
\DoxyCodeLine{64 \textcolor{comment}{  gzip wrapper, documented in RFC 1952, wrapped around a deflate stream.}}
\DoxyCodeLine{65 \textcolor{comment}{}}
\DoxyCodeLine{66 \textcolor{comment}{     This library can optionally read and write gzip streams in memory as well.}}
\DoxyCodeLine{67 \textcolor{comment}{}}
\DoxyCodeLine{68 \textcolor{comment}{     The zlib format was designed to be compact and fast for use in memory}}
\DoxyCodeLine{69 \textcolor{comment}{  and on communications channels.  The gzip format was designed for single-\/}}
\DoxyCodeLine{70 \textcolor{comment}{  file compression on file systems, has a larger header than zlib to maintain}}
\DoxyCodeLine{71 \textcolor{comment}{  directory information, and uses a different, slower check method than zlib.}}
\DoxyCodeLine{72 \textcolor{comment}{}}
\DoxyCodeLine{73 \textcolor{comment}{     The library does not install any signal handler. The decoder checks}}
\DoxyCodeLine{74 \textcolor{comment}{  the consistency of the compressed data, so the library should never}}
\DoxyCodeLine{75 \textcolor{comment}{  crash even in case of corrupted input.}}
\DoxyCodeLine{76 \textcolor{comment}{*/}}
