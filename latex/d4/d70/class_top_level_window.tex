\doxysubsection{Detailed Description}
A base class for top-\/level windows.

This class is used for components that are considered a major part of your application -\/ e.\+g. \mbox{\hyperlink{class_resizable_window}{Resizable\+Window}}, \mbox{\hyperlink{class_document_window}{Document\+Window}}, \mbox{\hyperlink{class_dialog_window}{Dialog\+Window}}, \mbox{\hyperlink{class_alert_window}{Alert\+Window}}, etc. Things like menus that pop up briefly aren\textquotesingle{}t derived from it.

A \mbox{\hyperlink{class_top_level_window}{Top\+Level\+Window}} is probably on the desktop, but this isn\textquotesingle{}t mandatory -\/ it could itself be the child of another component.

The class manages a list of all instances of top-\/level windows that are in use, and each one is also given the concept of being \char`\"{}active\char`\"{}. The active window is one that is actively being used by the user. This isn\textquotesingle{}t quite the same as the component with the keyboard focus, because there may be a popup menu or other temporary window which gets keyboard focus while the active top level window is unchanged.

A top-\/level window also has an optional drop-\/shadow.

Makes this component appear as a window on the desktop.

Note that before calling this, you should make sure that the component\textquotesingle{}s opacity is set correctly using \mbox{\hyperlink{class_component_a7320d543cba40183c894474ab78798ea}{set\+Opaque()}}. If the component is non-\/opaque, the windowing system will try to create a special transparent window for it, which will generally take a lot more CPU to operate (and might not even be possible on some platforms).

If the component is inside a parent component at the time this method is called, it will first be removed from that parent. Likewise if a component is on the desktop and is subsequently added to another component, it\textquotesingle{}ll be removed from the desktop.

Called to indicate a change in whether or not this component is the parent of the currently-\/focused component.

Essentially this is called when the return value of a call to has\+Keyboard\+Focus (true) has changed. It happens when focus moves from one of this component\textquotesingle{}s children (at any depth) to a component that isn\textquotesingle{}t contained in this one, (or vice-\/versa). Note that this method does NOT get called to when focus simply moves from one of its child components to another.
