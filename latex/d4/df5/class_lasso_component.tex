\doxysubsection{Detailed Description}
\subsubsection*{template$<$class Selectable\+Item\+Type$>$\newline
class Lasso\+Component$<$ Selectable\+Item\+Type $>$}
A component that acts as a rectangular selection region, which you drag with the mouse to select groups of objects (in conjunction with a \mbox{\hyperlink{class_selected_item_set}{Selected\+Item\+Set}}).

To use one of these\+:


\begin{DoxyItemize}
\item In your mouse\+Down or mouse\+Drag event, add the \mbox{\hyperlink{class_lasso_component}{Lasso\+Component}} to your parent component, and call its \mbox{\hyperlink{class_lasso_component_af9d043865ee8f7db2989e7200d13dfd2}{begin\+Lasso()}} method, giving it a suitable \mbox{\hyperlink{class_lasso_source}{Lasso\+Source}} object that it can use to find out which items are in the active area.
\item Each time your parent component gets a mouse\+Drag event, call \mbox{\hyperlink{class_lasso_component_a4aca7f9f28ecef2834bfabe648ba6959}{drag\+Lasso()}} to update the lasso\textquotesingle{}s position -\/ it will use its \mbox{\hyperlink{class_lasso_source}{Lasso\+Source}} to calculate and update the current selection.
\item After the drag has finished and you get a mouse\+Up callback, you should call \mbox{\hyperlink{class_lasso_component_a77e7463330941cd3ef81fa41209387f0}{end\+Lasso()}} to clean up. This will make the lasso component invisible, and you can remove it from the parent component, or delete it.
\end{DoxyItemize}

The class takes into account the modifier keys that are being held down while the lasso is being dragged, so if shift is pressed, then any lassoed items will be added to the original selection; if ctrl or command is pressed, they will be xor\textquotesingle{}ed with any previously selected items.

Tests whether a given point is inside the component.

Overriding this method allows you to create components which only intercept mouse-\/clicks within a user-\/defined area.

This is called to find out whether a particular x, y coordinate is considered to be inside the component or not, and is used by methods such as \mbox{\hyperlink{class_component_a6b697f6caaa753756251b4eb5ece302b}{contains()}} and \mbox{\hyperlink{class_component_a054f5a6add8f7dd6ce7cd088bc5e579f}{get\+Component\+At()}} to work out which component the mouse is clicked on.

Components with custom shapes will probably want to override it to perform some more complex hit-\/testing.

The default implementation of this method returns either true or false, depending on the value that was set by calling \mbox{\hyperlink{class_component_ac9fdcb595c1878201a641de2e1159aec}{set\+Intercepts\+Mouse\+Clicks()}} (true is the default return value).

Note that the hit-\/test region is not related to the opacity with which areas of a component are painted.

Applications should never call \mbox{\hyperlink{class_lasso_component_ab67925a4436687bc41437c8460de710d}{hit\+Test()}} directly -\/ instead use the \mbox{\hyperlink{class_component_a6b697f6caaa753756251b4eb5ece302b}{contains()}} method, because this will also test for occlusion by the component\textquotesingle{}s parent.

Note that for components on the desktop, this method will be ignored, because it\textquotesingle{}s not always possible to implement this behaviour on all platforms.

Components can override this method to draw their content.

The \mbox{\hyperlink{class_lasso_component_a0d9415c5dd551e76336ca1d952d5f6f9}{paint()}} method gets called when a region of a component needs redrawing, either because the component\textquotesingle{}s \mbox{\hyperlink{class_component_af3ab3eabec93dd60faf983370f3a1206}{repaint()}} method has been called, or because something has happened on the screen that means a section of a window needs to be redrawn.

Any child components will draw themselves over whatever this method draws. If you need to paint over the top of your child components, you can also implement the \mbox{\hyperlink{class_component_aa15dc77c06b5a0a4192090b4a323f79c}{paint\+Over\+Children()}} method to do this.

If you want to cause a component to redraw itself, this is done asynchronously -\/ calling the \mbox{\hyperlink{class_component_af3ab3eabec93dd60faf983370f3a1206}{repaint()}} method marks a region of the component as \char`\"{}dirty\char`\"{}, and the \mbox{\hyperlink{class_lasso_component_a0d9415c5dd551e76336ca1d952d5f6f9}{paint()}} method will automatically be called sometime later, by the message thread, to paint any bits that need refreshing. In JUCE (and almost all modern UI frameworks), you never redraw something synchronously.

You should never need to call this method directly -\/ to take a snapshot of the component you could use \mbox{\hyperlink{class_component_a275e46e5a8147f89c142c91ba5c793a5}{create\+Component\+Snapshot()}} or \mbox{\hyperlink{class_component_a2c960bd5ec5ac76d18e82b6883096714}{paint\+Entire\+Component()}}.
