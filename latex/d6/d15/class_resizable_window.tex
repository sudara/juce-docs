\doxysubsection{Detailed Description}
A base class for top-\/level windows that can be dragged around and resized.

To add content to the window, use its \mbox{\hyperlink{class_resizable_window_a97940f07b6014bac018cbe9330abc769}{set\+Content\+Owned()}} or \mbox{\hyperlink{class_resizable_window_ad6c44adc91ca926e9c0a6ecb4cf1bfa7}{set\+Content\+Non\+Owned()}} methods to give it a component that will remain positioned inside it (leaving a gap around the edges for a border).

It\textquotesingle{}s not advisable to add child components directly to a \mbox{\hyperlink{class_resizable_window}{Resizable\+Window}}\+: put them inside your content component instead. And overriding methods like \mbox{\hyperlink{class_resizable_window_ae42aa18209b1d14e8441dce5b991cb07}{resized()}}, \mbox{\hyperlink{class_resizable_window_acfb2ced15e476e78f9a71b0214414798}{moved()}}, etc is also not recommended -\/ instead override these methods for your content component. (If for some obscure reason you do need to override these methods, always remember to call the super-\/class\textquotesingle{}s \mbox{\hyperlink{class_resizable_window_ae42aa18209b1d14e8441dce5b991cb07}{resized()}} method too, otherwise it\textquotesingle{}ll fail to lay out the window decorations correctly).

By default resizing isn\textquotesingle{}t enabled -\/ use the \mbox{\hyperlink{class_resizable_window_a538709f0111ad32b05d858a511bcd5be}{set\+Resizable()}} method to enable it and to choose the style of resizing to use.

Makes this component appear as a window on the desktop.

Note that before calling this, you should make sure that the component\textquotesingle{}s opacity is set correctly using \mbox{\hyperlink{class_component_a7320d543cba40183c894474ab78798ea}{set\+Opaque()}}. If the component is non-\/opaque, the windowing system will try to create a special transparent window for it, which will generally take a lot more CPU to operate (and might not even be possible on some platforms).

If the component is inside a parent component at the time this method is called, it will first be removed from that parent. Likewise if a component is on the desktop and is subsequently added to another component, it\textquotesingle{}ll be removed from the desktop.

Components can override this method to draw their content.

The \mbox{\hyperlink{class_resizable_window_a97b64ce86b466f26b489f533946b42cc}{paint()}} method gets called when a region of a component needs redrawing, either because the component\textquotesingle{}s \mbox{\hyperlink{class_component_af3ab3eabec93dd60faf983370f3a1206}{repaint()}} method has been called, or because something has happened on the screen that means a section of a window needs to be redrawn.

Any child components will draw themselves over whatever this method draws. If you need to paint over the top of your child components, you can also implement the \mbox{\hyperlink{class_component_aa15dc77c06b5a0a4192090b4a323f79c}{paint\+Over\+Children()}} method to do this.

If you want to cause a component to redraw itself, this is done asynchronously -\/ calling the \mbox{\hyperlink{class_component_af3ab3eabec93dd60faf983370f3a1206}{repaint()}} method marks a region of the component as \char`\"{}dirty\char`\"{}, and the \mbox{\hyperlink{class_resizable_window_a97b64ce86b466f26b489f533946b42cc}{paint()}} method will automatically be called sometime later, by the message thread, to paint any bits that need refreshing. In JUCE (and almost all modern UI frameworks), you never redraw something synchronously.

You should never need to call this method directly -\/ to take a snapshot of the component you could use \mbox{\hyperlink{class_component_a275e46e5a8147f89c142c91ba5c793a5}{create\+Component\+Snapshot()}} or \mbox{\hyperlink{class_component_a2c960bd5ec5ac76d18e82b6883096714}{paint\+Entire\+Component()}}.
