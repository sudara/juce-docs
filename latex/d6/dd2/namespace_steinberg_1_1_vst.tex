{\footnotesize\ttfamily enum \mbox{\hyperlink{namespace_steinberg_1_1_vst_a6dc0ab53a66ce6b1e2dc995ff8aaf1ee}{Steinberg\+::\+Vst\+::\+Process\+Modes}}}

Processing mode informs the plug-\/in about the context and at which frequency the process call is called. VST3 defines 3 modes\+:
\begin{DoxyItemize}
\item k\+Realtime\+: each process call is called at a realtime frequency (defined by \mbox{[}num\+Samples of \mbox{\hyperlink{struct_steinberg_1_1_vst_1_1_process_data}{Process\+Data}}\mbox{]} / samplerate). The plug-\/in should always try to process as fast as possible in order to let enough time slice to other plug-\/ins.
\item k\+Prefetch\+: each process call could be called at a variable frequency (jitter, slower / faster than realtime), the plug-\/in should process at the same quality level than realtime, plug-\/in must not slow down to realtime (e.\+g. disk streaming)! The host should avoid to process in k\+Prefetch mode such sampler based plug-\/in.
\item k\+Offline\+: each process call could be faster than realtime or slower, higher quality than realtime could be used. plug-\/ins using disk streaming should be sure that they have enough time in the process call for streaming, if needed by slowing down to realtime or slower.
\end{DoxyItemize}Note about \mbox{\hyperlink{class_process}{Process}} Modes switching\+:
\begin{DoxyItemize}
\item Switching between k\+Realtime and k\+Prefetch process modes are done in realtime thread without need of calling \mbox{\hyperlink{class_steinberg_1_1_vst_1_1_i_audio_processor_a1ccb3d6e5d91a6b62a634011bcf0a48b}{IAudio\+Processor\+::setup\+Processing}}, the plug-\/in should check in process call the member process\+Mode of \mbox{\hyperlink{struct_steinberg_1_1_vst_1_1_process_data}{Process\+Data}} in order to know in which mode it is processed.
\item Switching between k\+Realtime (or k\+Prefetch) and k\+Offline requires that the host calls \mbox{\hyperlink{class_steinberg_1_1_vst_1_1_i_audio_processor_a1ccb3d6e5d91a6b62a634011bcf0a48b}{IAudio\+Processor\+::setup\+Processing}} in order to inform the plug-\/in about this mode change.
