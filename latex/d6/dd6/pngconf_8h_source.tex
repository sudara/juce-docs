\DoxyCodeLine{97 \textcolor{comment}{/* Function calling conventions.}}
\DoxyCodeLine{98 \textcolor{comment}{ * =============================}}
\DoxyCodeLine{99 \textcolor{comment}{ * Normally it is not necessary to specify to the compiler how to call}}
\DoxyCodeLine{100 \textcolor{comment}{ * a function -\/ it just does it -\/ however on x86 systems derived from}}
\DoxyCodeLine{101 \textcolor{comment}{ * Microsoft and Borland C compilers ('IBM PC', 'DOS', 'Windows' systems}}
\DoxyCodeLine{102 \textcolor{comment}{ * and some others) there are multiple ways to call a function and the}}
\DoxyCodeLine{103 \textcolor{comment}{ * default can be changed on the compiler command line.  For this reason}}
\DoxyCodeLine{104 \textcolor{comment}{ * libpng specifies the calling convention of every exported function and}}
\DoxyCodeLine{105 \textcolor{comment}{ * every function called via a user supplied function pointer.  This is}}
\DoxyCodeLine{106 \textcolor{comment}{ * done in this file by defining the following macros:}}
\DoxyCodeLine{107 \textcolor{comment}{ *}}
\DoxyCodeLine{108 \textcolor{comment}{ * PNGAPI    Calling convention for exported functions.}}
\DoxyCodeLine{109 \textcolor{comment}{ * PNGCBAPI  Calling convention for user provided (callback) functions.}}
\DoxyCodeLine{110 \textcolor{comment}{ * PNGCAPI   Calling convention used by the ANSI-\/C library (required}}
\DoxyCodeLine{111 \textcolor{comment}{ *           for longjmp callbacks and sometimes used internally to}}
\DoxyCodeLine{112 \textcolor{comment}{ *           specify the calling convention for zlib).}}
\DoxyCodeLine{113 \textcolor{comment}{ *}}
\DoxyCodeLine{114 \textcolor{comment}{ * These macros should never be overridden.  If it is necessary to}}
\DoxyCodeLine{115 \textcolor{comment}{ * change calling convention in a private build this can be done}}
\DoxyCodeLine{116 \textcolor{comment}{ * by setting PNG\_API\_RULE (which defaults to 0) to one of the values}}
\DoxyCodeLine{117 \textcolor{comment}{ * below to select the correct 'API' variants.}}
\DoxyCodeLine{118 \textcolor{comment}{ *}}
\DoxyCodeLine{119 \textcolor{comment}{ * PNG\_API\_RULE=0 Use PNGCAPI -\/ the 'C' calling convention -\/ throughout.}}
\DoxyCodeLine{120 \textcolor{comment}{ *                This is correct in every known environment.}}
\DoxyCodeLine{121 \textcolor{comment}{ * PNG\_API\_RULE=1 Use the operating system convention for PNGAPI and}}
\DoxyCodeLine{122 \textcolor{comment}{ *                the 'C' calling convention (from PNGCAPI) for}}
\DoxyCodeLine{123 \textcolor{comment}{ *                callbacks (PNGCBAPI).  This is no longer required}}
\DoxyCodeLine{124 \textcolor{comment}{ *                in any known environment -\/ if it has to be used}}
\DoxyCodeLine{125 \textcolor{comment}{ *                please post an explanation of the problem to the}}
\DoxyCodeLine{126 \textcolor{comment}{ *                libpng mailing list.}}
\DoxyCodeLine{127 \textcolor{comment}{ *}}
\DoxyCodeLine{128 \textcolor{comment}{ * These cases only differ if the operating system does not use the C}}
\DoxyCodeLine{129 \textcolor{comment}{ * calling convention, at present this just means the above cases}}
\DoxyCodeLine{130 \textcolor{comment}{ * (x86 DOS/Windows systems) and, even then, this does not apply to}}
\DoxyCodeLine{131 \textcolor{comment}{ * Cygwin running on those systems.}}

\DoxyCodeLine{134 \textcolor{comment}{ * the application uses to call the library matches the conventions}}
\DoxyCodeLine{135 \textcolor{comment}{ * set when building the library.}}
\DoxyCodeLine{136 \textcolor{comment}{ */}}

\DoxyCodeLine{138 \textcolor{comment}{/* Symbol export}}
\DoxyCodeLine{139 \textcolor{comment}{ * =============}}
\DoxyCodeLine{140 \textcolor{comment}{ * When building a shared library it is almost always necessary to tell}}
\DoxyCodeLine{141 \textcolor{comment}{ * the compiler which symbols to export.  The png.h macro 'PNG\_EXPORT'}}
\DoxyCodeLine{142 \textcolor{comment}{ * is used to mark the symbols.  On some systems these symbols can be}}
\DoxyCodeLine{143 \textcolor{comment}{ * extracted at link time and need no special processing by the compiler,}}
\DoxyCodeLine{144 \textcolor{comment}{ * on other systems the symbols are flagged by the compiler and just}}
\DoxyCodeLine{145 \textcolor{comment}{ * the declaration requires a special tag applied (unfortunately) in a}}
\DoxyCodeLine{146 \textcolor{comment}{ * compiler dependent way.  Some systems can do either.}}
\DoxyCodeLine{147 \textcolor{comment}{ *}}
\DoxyCodeLine{148 \textcolor{comment}{ * A small number of older systems also require a symbol from a DLL to}}
\DoxyCodeLine{149 \textcolor{comment}{ * be flagged to the program that calls it.  This is a problem because}}
\DoxyCodeLine{150 \textcolor{comment}{ * we do not know in the header file included by application code that}}
\DoxyCodeLine{151 \textcolor{comment}{ * the symbol will come from a shared library, as opposed to a statically}}
\DoxyCodeLine{152 \textcolor{comment}{ * linked one.  For this reason the application must tell us by setting}}
\DoxyCodeLine{153 \textcolor{comment}{ * the magic flag PNG\_USE\_DLL to turn on the special processing before}}
\DoxyCodeLine{154 \textcolor{comment}{ * it includes png.h.}}
\DoxyCodeLine{155 \textcolor{comment}{ *}}
\DoxyCodeLine{156 \textcolor{comment}{ * Four additional macros are used to make this happen:}}
\DoxyCodeLine{157 \textcolor{comment}{ *}}
\DoxyCodeLine{158 \textcolor{comment}{ * PNG\_IMPEXP The magic (if any) to cause a symbol to be exported from}}
\DoxyCodeLine{159 \textcolor{comment}{ *            the build or imported if PNG\_USE\_DLL is set -\/ compiler}}
\DoxyCodeLine{160 \textcolor{comment}{ *            and system specific.}}
\DoxyCodeLine{161 \textcolor{comment}{ *}}
\DoxyCodeLine{162 \textcolor{comment}{ * PNG\_EXPORT\_TYPE(type) A macro that pre or appends PNG\_IMPEXP to}}
\DoxyCodeLine{163 \textcolor{comment}{ *                       'type', compiler specific.}}
\DoxyCodeLine{164 \textcolor{comment}{ *}}
\DoxyCodeLine{165 \textcolor{comment}{ * PNG\_DLL\_EXPORT Set to the magic to use during a libpng build to}}
\DoxyCodeLine{166 \textcolor{comment}{ *                make a symbol exported from the DLL.  Not used in the}}
\DoxyCodeLine{167 \textcolor{comment}{ *                public header files; see pngpriv.h for how it is used}}
\DoxyCodeLine{168 \textcolor{comment}{ *                in the libpng build.}}
\DoxyCodeLine{169 \textcolor{comment}{ *}}
\DoxyCodeLine{170 \textcolor{comment}{ * PNG\_DLL\_IMPORT Set to the magic to force the libpng symbols to come}}
