By default it adds the cut/copy/paste items, but you can override this if you need to replace these with your own items.

If you want to add your own items to the existing ones, you can override this, call the base class\textquotesingle{}s \mbox{\hyperlink{class_text_editor_a5887eae43a0517c2c986d54b0f02e9bd}{add\+Popup\+Menu\+Items()}} method, then append your own items.

When the menu has been shown, \mbox{\hyperlink{class_text_editor_ac90236dab0312511ba8bc38b2507a575}{perform\+Popup\+Menu\+Action()}} will be called to perform the item that the user has chosen.

The default menu items will be added using item IDs from the \mbox{\hyperlink{namespace_standard_application_command_i_ds}{Standard\+Application\+Command\+IDs}} namespace.

If this was triggered by a mouse-\/click, the mouse\+Click\+Event parameter will be a pointer to the info about it, or may be null if the menu is being triggered by some other means.

Called when a key is pressed or released.

Whenever a key on the keyboard is pressed or released (including modifier keys like shift and ctrl), this method will be called on the component that currently has the keyboard focus. Remember that a component will only be given the focus if its \mbox{\hyperlink{class_component_a6a1f21a76b971d54870cb5c32c041055}{set\+Wants\+Keyboard\+Focus()}} method has been used to enable this.

If your implementation returns true, the event will be consumed and not passed on to any other listeners. If it returns false, then any Key\+Listeners that have been registered with this component will have their key\+State\+Changed methods called. As soon as one of these returns true, the process will stop, but if they all return false, the event will be passed upwards to this component\textquotesingle{}s parent, and so on.

The default implementation of this method does nothing and returns false.

To find out which keys are up or down at any time, see the \mbox{\hyperlink{class_key_press_a22dc8e2e91023e57ef9e102aef0c6961}{Key\+Press\+::is\+Key\+Currently\+Down()}} method.

Components can override this method to draw their content.

The \mbox{\hyperlink{class_text_editor_a8b7485621bf88d75d8e51c01f9d2b82c}{paint()}} method gets called when a region of a component needs redrawing, either because the component\textquotesingle{}s \mbox{\hyperlink{class_component_af3ab3eabec93dd60faf983370f3a1206}{repaint()}} method has been called, or because something has happened on the screen that means a section of a window needs to be redrawn.

Any child components will draw themselves over whatever this method draws. If you need to paint over the top of your child components, you can also implement the \mbox{\hyperlink{class_text_editor_a2a8635846ebc9fd09bc42a256b3480f4}{paint\+Over\+Children()}} method to do this.

If you want to cause a component to redraw itself, this is done asynchronously -\/ calling the \mbox{\hyperlink{class_component_af3ab3eabec93dd60faf983370f3a1206}{repaint()}} method marks a region of the component as \char`\"{}dirty\char`\"{}, and the \mbox{\hyperlink{class_text_editor_a8b7485621bf88d75d8e51c01f9d2b82c}{paint()}} method will automatically be called sometime later, by the message thread, to paint any bits that need refreshing. In JUCE (and almost all modern UI frameworks), you never redraw something synchronously.

You should never need to call this method directly -\/ to take a snapshot of the component you could use \mbox{\hyperlink{class_component_a275e46e5a8147f89c142c91ba5c793a5}{create\+Component\+Snapshot()}} or \mbox{\hyperlink{class_component_a2c960bd5ec5ac76d18e82b6883096714}{paint\+Entire\+Component()}}.
