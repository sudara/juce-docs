{\footnotesize\ttfamily bool juce\+::\+Toolbar\+::\+Customisation\+Dialog\+::can\+Modal\+Event\+Be\+Sent\+To\+Component (\begin{DoxyParamCaption}\item[{\mbox{\hyperlink{zconf_8in_8h_a2c212835823e3c54a8ab6d95c652660e}{const}} \mbox{\hyperlink{class_component}{Component}} $\ast$}]{target\+Component }\end{DoxyParamCaption})\hspace{0.3cm}{\ttfamily [inline]}, {\ttfamily [override]}, {\ttfamily [virtual]}}

When a component is modal, this callback allows it to choose which other components can still receive events.

When a modal component is active and the user clicks on a non-\/modal component, this method is called on the modal component, and if it returns true, the event is allowed to reach its target. If it returns false, the event is blocked and the \mbox{\hyperlink{class_component_a260e46fb72d4b3210bc46a658a21fbb5}{input\+Attempt\+When\+Modal()}} callback is made.

It called by the \mbox{\hyperlink{class_component_a70fc33489ec892d615452dfdbd4687db}{is\+Currently\+Blocked\+By\+Another\+Modal\+Component()}} method. The default implementation just returns false in all cases.

{\footnotesize\ttfamily \mbox{\hyperlink{png_8h_aa8c59027f9ab2769342f248709d68d17}{void}} juce\+::\+Toolbar\+::\+Customisation\+Dialog\+::close\+Button\+Pressed (\begin{DoxyParamCaption}{ }\end{DoxyParamCaption})\hspace{0.3cm}{\ttfamily [inline]}, {\ttfamily [override]}, {\ttfamily [virtual]}}

This method is called when the user tries to close the window.

This is triggered by the user clicking the close button, or using some other OS-\/specific key shortcut or OS menu for getting rid of a window.

If the window is just a pop-\/up, you should override this \mbox{\hyperlink{classjuce_1_1_toolbar_1_1_customisation_dialog_a763122d9687c03faf3143c6c33af05a6}{close\+Button\+Pressed()}} method and make it delete the window in whatever way is appropriate for your app. E.\+g. you might just want to call \char`\"{}delete this\char`\"{}.

If your app is centred around this window such that the whole app should quit when the window is closed, then you will probably want to use this method as an opportunity to call \mbox{\hyperlink{class_j_u_c_e_application_base_a542e212605f0aa693f687d8798998615}{JUCEApplication\+Base\+::quit()}}, and leave the window to be deleted later by your \mbox{\hyperlink{class_j_u_c_e_application_base_a318021f86bf379caeb4cb27c32f0f229}{JUCEApplication\+Base\+::shutdown()}} method. (Doing it this way means that your window will still get cleaned-\/up if the app is quit by some other means (e.\+g. a cmd-\/Q on the mac or closing it via the taskbar icon on Windows).

(Note that the \mbox{\hyperlink{class_document_window}{Document\+Window}} class overrides \mbox{\hyperlink{class_component_a543e70d5a3ff75cc3facb476f4cdb131}{Component\+::user\+Tried\+To\+Close\+Window()}} and redirects it to call this method, so any methods of closing the window that are caught by \mbox{\hyperlink{class_document_window_a056defe79da104beb80243a1416c1f8b}{user\+Tried\+To\+Close\+Window()}} will also end up here).
