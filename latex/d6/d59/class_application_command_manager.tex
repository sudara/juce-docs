\doxysubsection{Detailed Description}
One of these objects holds a list of all the commands your app can perform, and despatches these commands when needed.

Application commands are a good way to trigger actions in your app, e.\+g. \char`\"{}\+Quit\char`\"{}, \char`\"{}\+Copy\char`\"{}, \char`\"{}\+Paste\char`\"{}, etc. Menus, buttons and keypresses can all be given commands to invoke automatically, which means you don\textquotesingle{}t have to handle the result of a menu or button click manually. Commands are despatched to \mbox{\hyperlink{class_application_command_target}{Application\+Command\+Target}} objects which can choose which events they want to handle.

This architecture also allows for nested Application\+Command\+Targets, so that for example you could have two different objects, one inside the other, both of which can respond to a \char`\"{}delete\char`\"{} command. Depending on which one has focus, the command will be sent to the appropriate place, regardless of whether it was triggered by a menu, keypress or some other method.

To set up your app to use commands, you\textquotesingle{}ll need to do the following\+:


\begin{DoxyItemize}
\item Create a global \mbox{\hyperlink{class_application_command_manager}{Application\+Command\+Manager}} to hold the list of all possible commands. (This will also manage a set of key-\/mappings for them).
\item Make some of your UI components (or other objects) inherit from \mbox{\hyperlink{class_application_command_target}{Application\+Command\+Target}}. This allows the object to provide a list of commands that it can perform, and to handle them.
\item Register each type of command using \mbox{\hyperlink{class_application_command_manager_ac3f8c831583499556579848779e030cf}{Application\+Command\+Manager\+::register\+All\+Commands\+For\+Target()}}, or \mbox{\hyperlink{class_application_command_manager_a151be8c30a8339675ad5dcba5a5c4e62}{Application\+Command\+Manager\+::register\+Command()}}.
\item If you want key-\/presses to trigger your commands, use the \mbox{\hyperlink{class_application_command_manager_acaab9b68b87978372c18a29d18864892}{Application\+Command\+Manager\+::get\+Key\+Mappings()}} method to access the key-\/mapper object, which you will need to register as a key-\/listener in whatever top-\/level component you\textquotesingle{}re using. See the \mbox{\hyperlink{class_key_press_mapping_set}{Key\+Press\+Mapping\+Set}} class for more help about setting this up.
\item Use methods such as \mbox{\hyperlink{class_popup_menu_a3484289881ee2c045bcf3e4566473880}{Popup\+Menu\+::add\+Command\+Item()}} or \mbox{\hyperlink{class_button_a3d29d9c9ab75389a170b67a364e0cb9e}{Button\+::set\+Command\+To\+Trigger()}} to cause these commands to be invoked automatically.
\item Commands can be invoked directly by your code using \mbox{\hyperlink{class_application_command_manager_a55f04432929174c2de6d622cb2d23c67}{Application\+Command\+Manager\+::invoke\+Directly()}}.
\end{DoxyItemize}

When a command is invoked, the \mbox{\hyperlink{class_application_command_manager}{Application\+Command\+Manager}} will try to choose the best \mbox{\hyperlink{class_application_command_target}{Application\+Command\+Target}} to receive the specified command. To do this it will use the current keyboard focus to see which component might be interested, and will search the component hierarchy for those that also implement the \mbox{\hyperlink{class_application_command_target}{Application\+Command\+Target}} interface. If an \mbox{\hyperlink{class_application_command_target}{Application\+Command\+Target}} isn\textquotesingle{}t interested in the command that is being invoked, then the next one in line will be tried (see the \mbox{\hyperlink{class_application_command_target_a8def2ea10aad1a34d37f9afe1675b06d}{Application\+Command\+Target\+::get\+Next\+Command\+Target()}} method), and so on until \mbox{\hyperlink{class_application_command_target_a8def2ea10aad1a34d37f9afe1675b06d}{Application\+Command\+Target\+::get\+Next\+Command\+Target()}} returns nullptr. At this point if the command still hasn\textquotesingle{}t been performed, it will be passed to the current \mbox{\hyperlink{class_j_u_c_e_application}{JUCEApplication}} object (which is itself an \mbox{\hyperlink{class_application_command_target}{Application\+Command\+Target}}).

To exert some custom control over which \mbox{\hyperlink{class_application_command_target}{Application\+Command\+Target}} is chosen to invoke a command, you can override the \mbox{\hyperlink{class_application_command_manager_af8fb9732eaddb5f8df844f3915f0e914}{Application\+Command\+Manager\+::get\+First\+Command\+Target()}} method and choose the object yourself.

This should be called to tell the manager that one of its registered commands may have changed its active status.

Because the command manager only finds out whether a command is active or inactive by querying the current \mbox{\hyperlink{class_application_command_target}{Application\+Command\+Target}}, this is used to tell it that things may have changed. Chooses the \mbox{\hyperlink{class_application_command_target}{Application\+Command\+Target}} to which a command should be sent.

Whenever the manager needs to know which target a command should be sent to, it calls this method to determine the first one to try.

By default, this method will return the target that was set by calling \mbox{\hyperlink{class_application_command_manager_abebbbdd281b83f02a94c909f93b5347f}{set\+First\+Command\+Target()}}. If no target is set, it will return the result of \mbox{\hyperlink{class_application_command_manager_a0b7dd4ce5b031913d508a9e780f88282}{find\+Default\+Component\+Target()}}.

If you need to make sure all commands go via your own custom target, then you can either use \mbox{\hyperlink{class_application_command_manager_abebbbdd281b83f02a94c909f93b5347f}{set\+First\+Command\+Target()}} to specify a single target, or override this method if you need more complex logic to choose one.

It may return nullptr if no targets are available.

Tries to find the best target to use to perform a given command.

This will call \mbox{\hyperlink{class_application_command_manager_af8fb9732eaddb5f8df844f3915f0e914}{get\+First\+Command\+Target()}} to find the preferred target, and will check whether that target can handle the given command. If it can\textquotesingle{}t, then it\textquotesingle{}ll use \mbox{\hyperlink{class_application_command_target_a8def2ea10aad1a34d37f9afe1675b06d}{Application\+Command\+Target\+::get\+Next\+Command\+Target()}} to find the next one to try, and so on until no more are available.

If no targets are found that can perform the command, this method will return nullptr.

If a target is found, then it will get the target to fill-\/in the up\+To\+Date\+Info structure with the latest info about that command, so that the caller can see whether the command is disabled, ticked, etc. \mbox{\Hypertarget{class_application_command_manager_a6b506331c1cfb3d07b2925ac16b8141e}\label{class_application_command_manager_a6b506331c1cfb3d07b2925ac16b8141e}}
