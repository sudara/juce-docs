This is used to create or update a custom component to go in a row of the list.

Any row may contain a custom component, or can just be drawn with the \mbox{\hyperlink{class_list_box_model_a255cf47783413c4f3fa71008a34cb776}{paint\+List\+Box\+Item()}} method and handle mouse clicks with \mbox{\hyperlink{class_list_box_model_a0c8da1d24d4de91660fc4878436f22c6}{list\+Box\+Item\+Clicked()}}.

This method will be called whenever a custom component might need to be updated -\/ e.\+g. when the list is changed, or \mbox{\hyperlink{class_list_box_a3e8cb20434a462c1a102cf62e112e16d}{List\+Box\+::update\+Content()}} is called.

If you don\textquotesingle{}t need a custom component for the specified row, then return nullptr. (Bear in mind that even if you\textquotesingle{}re not creating a new component, you may still need to delete existing\+Component\+To\+Update if it\textquotesingle{}s non-\/null).

If you do want a custom component, and the existing\+Component\+To\+Update is null, then this method must create a suitable new component and return it.

If the existing\+Component\+To\+Update is non-\/null, it will be a pointer to a component previously created by this method. In this case, the method must either update it to make sure it\textquotesingle{}s correctly representing the given row (which may be different from the one that the component was created for), or it can delete this component and return a new one.

The component that your method returns will be deleted by the \mbox{\hyperlink{class_list_box}{List\+Box}} when it is no longer needed.

Bear in mind that if you put a custom component inside the row but still want the listbox to automatically handle clicking, selection, etc, then you\textquotesingle{}ll need to make sure your custom component doesn\textquotesingle{}t intercept all the mouse events that land on it, e.\+g by using \mbox{\hyperlink{class_component_ac9fdcb595c1878201a641de2e1159aec}{Component\+::set\+Intercepts\+Mouse\+Clicks()}}.
