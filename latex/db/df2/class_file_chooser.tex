After creating one of these, use one of the browse\+For... methods to display it.

\hline
{\em initial\+File\+Or\+Directory} & the file or directory that should be selected when the dialog box opens. If this parameter is set to \mbox{\hyperlink{class_file}{File()}}, a sensible default directory will be used instead. When using native dialogs, not all platforms will actually select the file. For example, on mac\+OS, when initial\+File\+Or\+Directory is a file, only the parent directory of initial\+File\+Or\+Directory will be used as the initial directory of the native file chooser.\\
\hline
\end{DoxyParams}
Note\+: On i\+OS when saving a file, a user will not be able to change a file name, so it may be a good idea to include at least a valid file name in initial\+File\+Or\+Directory. When no filename is found, \char`\"{}\+Untitled\char`\"{} will be used.

Also, if you pass an already existing file on i\+OS, that file will be automatically copied to the destination chosen by user and if it can be previewed, its preview will be presented in the dialog too. You will still be able to write into this file copy, since its \mbox{\hyperlink{class_u_r_l}{URL}} will be returned by \mbox{\hyperlink{class_file_chooser_a5964a831e9d12cd53de3606240dfd4c9}{get\+URLResult()}}. This can be useful when you want to save e.\+g. an image, so that you can pass a (temporary) file with low quality preview and after the user picks the destination, you can write a high quality image into the copied file. If you create such a temporary file, you need to delete it yourself, once it is not needed anymore.

{\footnotesize\ttfamily \mbox{\hyperlink{class_file}{File}} File\+Chooser\+::get\+Result (\begin{DoxyParamCaption}{ }\end{DoxyParamCaption}) const}

Returns the last file that was chosen by one of the browse\+For methods.

After calling the appropriate browse\+For... method, this method lets you find out what file or directory they chose.

Note that the file returned is only valid if the browse method returned true (i.\+e. if the user pressed \textquotesingle{}ok\textquotesingle{} rather than cancelling).

On mobile platforms, the file browser may return a \mbox{\hyperlink{class_u_r_l}{URL}} instead of a local file. Therefore, on mobile platforms, you should call \mbox{\hyperlink{class_file_chooser_a5964a831e9d12cd53de3606240dfd4c9}{get\+URLResult()}} instead.

If you\textquotesingle{}re using a multiple-\/file select, then use the \mbox{\hyperlink{class_file_chooser_aa3b5575ed02badb9eb536e359599a7cc}{get\+Results()}} method instead, to obtain the list of all files chosen.

{\footnotesize\ttfamily \mbox{\hyperlink{class_u_r_l}{URL}} File\+Chooser\+::get\+URLResult (\begin{DoxyParamCaption}{ }\end{DoxyParamCaption}) const}

Returns the last document that was chosen by one of the browse\+For methods.

Use this method if you are using the \mbox{\hyperlink{class_file_chooser}{File\+Chooser}} on a mobile platform which may return a \mbox{\hyperlink{class_u_r_l}{URL}} to a remote document. If a local file is chosen then you can convert this file to a JUCE \mbox{\hyperlink{class_file}{File}} class via the \mbox{\hyperlink{class_u_r_l_a2aa2e8a4033cf040c4c6273dbf3f6cdc}{URL\+::get\+Local\+File}} method.

Note\+: On i\+OS you must use the returned \mbox{\hyperlink{class_u_r_l}{URL}} object directly (you are also allowed to copy-\/ or move-\/construct another \mbox{\hyperlink{class_u_r_l}{URL}} from the returned \mbox{\hyperlink{class_u_r_l}{URL}}), rather than just storing the path as a \mbox{\hyperlink{class_string}{String}} and then creating a new \mbox{\hyperlink{class_u_r_l}{URL}} from that \mbox{\hyperlink{class_string}{String}}. This is because the returned \mbox{\hyperlink{class_u_r_l}{URL}} contains internally a security bookmark that is required to access the files pointed by it. Then, once you stop dealing with the file pointed by the \mbox{\hyperlink{class_u_r_l}{URL}}, you should dispose that \mbox{\hyperlink{class_u_r_l}{URL}} object, so that the security bookmark can be released by the system (only a limited number of such URLs is allowed).

Use this method to launch the file browser window asynchronously.

This will create a file browser dialog based on the settings in this structure and will launch it modally, returning immediately.

You must specify a callback which is called when the file browser is cancelled or a file is selected. To abort the file selection, simply delete the \mbox{\hyperlink{class_file_chooser}{File\+Chooser}} object.

You must ensure that the lifetime of the callback object is longer than the lifetime of the file-\/chooser.
