\begin{DoxyParagraph}{Sizing of a view}
Usually, the size of a plug-\/in view is fixed. But both the host and the plug-\/in can cause a view to be resized\+: ~\newline

\begin{DoxyItemize}
\item {\bfseries{Host\+:}} If \mbox{\hyperlink{class_steinberg_1_1_i_plug_view_a1d6056c4214e68553d625e39638958e9}{IPlug\+View\+::can\+Resize}} () returns k\+Result\+True the host will set up the window so that the user can resize it. While the user resizes the window, \mbox{\hyperlink{class_steinberg_1_1_i_plug_view_a80f4a909bca6fc2d270d0f5697cf0853}{IPlug\+View\+::check\+Size\+Constraint}} () is called, allowing the plug-\/in to change the size to a valid a valid supported rectangle size. The host then resizes the window to this rect and has to call \mbox{\hyperlink{class_steinberg_1_1_i_plug_view_a16b24ffce6c12f890ab854c7a7f77449}{IPlug\+View\+::on\+Size}} (). ~\newline
~\newline

\item {\bfseries{Plug-\/in}}\+: The plug-\/in can call \mbox{\hyperlink{class_steinberg_1_1_i_plug_frame_a5d20f1a8e68e38d9fd886c8cdab338fc}{IPlug\+Frame\+::resize\+View}} () and cause the host to resize the window.~\newline
~\newline
 Afterwards, in the same callstack, the host has to call \mbox{\hyperlink{class_steinberg_1_1_i_plug_view_a16b24ffce6c12f890ab854c7a7f77449}{IPlug\+View\+::on\+Size}} () if a resize is needed (size was changed). Note that if the host calls \mbox{\hyperlink{class_steinberg_1_1_i_plug_view_a7d4b424653a019f62ebd5632bfa7951b}{IPlug\+View\+::get\+Size}} () before calling \mbox{\hyperlink{class_steinberg_1_1_i_plug_view_a16b24ffce6c12f890ab854c7a7f77449}{IPlug\+View\+::on\+Size}} () (if needed), it will get the current (old) size not the wanted one!!~\newline
 Here the calling sequence\+:~\newline

\begin{DoxyItemize}
\item plug-\/in-\/\texorpdfstring{$>$}{>}host\+: \mbox{\hyperlink{class_steinberg_1_1_i_plug_frame_a5d20f1a8e68e38d9fd886c8cdab338fc}{IPlug\+Frame\+::resize\+View}} (new\+Size)
\item host-\/\texorpdfstring{$>$}{>}plug-\/in (optional)\+: \mbox{\hyperlink{class_steinberg_1_1_i_plug_view_a7d4b424653a019f62ebd5632bfa7951b}{IPlug\+View\+::get\+Size}} () returns the current\+Size (not the new\+Size!)
\item host-\/\texorpdfstring{$>$}{>}plug-\/in\+: if new\+Size is different from the current size\+: \mbox{\hyperlink{class_steinberg_1_1_i_plug_view_a16b24ffce6c12f890ab854c7a7f77449}{IPlug\+View\+::on\+Size}} (new\+Size)
\item host-\/\texorpdfstring{$>$}{>}plug-\/in (optional)\+: \mbox{\hyperlink{class_steinberg_1_1_i_plug_view_a7d4b424653a019f62ebd5632bfa7951b}{IPlug\+View\+::get\+Size}} () returns the new\+Size ~\newline
{\bfseries{Please only resize the platform representation of the view when \mbox{\hyperlink{class_steinberg_1_1_i_plug_view_a16b24ffce6c12f890ab854c7a7f77449}{IPlug\+View\+::on\+Size}} () is called.}}
\end{DoxyItemize}
\end{DoxyItemize}
\end{DoxyParagraph}
\begin{DoxyParagraph}{Keyboard handling}
The plug-\/in view receives keyboard events from the host. A view implementation must not handle keyboard events by the means of platform callbacks, but let the host pass them to the view. The host depends on a proper return value when \mbox{\hyperlink{class_steinberg_1_1_i_plug_view_aa9ecd932b885a42d30f1ca908141a64a}{IPlug\+View\+::on\+Key\+Down}} is called, otherwise the plug-\/in view may cause a malfunction of the host\textquotesingle{}s key command handling.
\end{DoxyParagraph}
