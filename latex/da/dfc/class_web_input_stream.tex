Wait until the first byte is ready for reading.

This method will attempt to connect to the \mbox{\hyperlink{class_u_r_l}{URL}} given in the constructor and block until the status code and all response headers have been received or an error has occurred.

Note that most methods will call \mbox{\hyperlink{class_web_input_stream_abe8920cdc15843c44228c267db3e85db}{connect()}} internally if they are called without an established connection. Therefore, it is not necessary to explicitly call connect unless you would like to use a custom listener.

After a successful call to \mbox{\hyperlink{class_web_input_stream_abe8920cdc15843c44228c267db3e85db}{connect()}}, \mbox{\hyperlink{class_web_input_stream_a0ee598b0627aacd5c63050e54c8fbf93}{get\+Response\+Headers()}}, \mbox{\hyperlink{class_web_input_stream_a443038879b96a8181b1cb5cb0d0db22f}{get\+Total\+Length()}} and \mbox{\hyperlink{class_web_input_stream_aa0e03a9176ed0cbbe72f56237371b9f2}{get\+Status\+Code()}} will all be non-\/blocking.

Tries to move the current read position of the stream.

The position is an absolute number of bytes from the stream\textquotesingle{}s start.

For a \mbox{\hyperlink{class_web_input_stream}{Web\+Input\+Stream}}, this method will fail if wanted\+Pos is smaller than the current position. If wanted\+Pos is greater than the current position, then calling \mbox{\hyperlink{class_web_input_stream_a5c452e8463fe81843daee042a12bb0d8}{set\+Position()}} is the same as calling \mbox{\hyperlink{class_web_input_stream_a2735a5061e4ad8de89179b29aa8336c2}{read()}}, i.\+e. the skipped data will still be downloaded, although skipped bytes will be discarded immediately.
