\doxysubsection{Detailed Description}
Base class for an MPE-\/compatible musical device that can play sounds.

This class extends \mbox{\hyperlink{struct_m_p_e_synthesiser_base}{MPESynthesiser\+Base}} by adding the concept of voices, each of which can play a sound triggered by a \mbox{\hyperlink{struct_m_p_e_note}{MPENote}} that can be modulated by MPE dimensions like pressure, pitchbend, and timbre, while the note is sounding.

To create a synthesiser, you\textquotesingle{}ll need to create a subclass of \mbox{\hyperlink{class_m_p_e_synthesiser_voice}{MPESynthesiser\+Voice}} which can play back one of these sounds at a time.

Then you can use the \mbox{\hyperlink{class_m_p_e_synthesiser_acd5a57224ed678670766c3f0e1c611ac}{add\+Voice()}} methods to give the synthesiser a set of voices it can use to play notes. If you only give it one voice it will be monophonic -\/ the more voices it has, the more polyphony it\textquotesingle{}ll have available.

Then repeatedly call the \mbox{\hyperlink{struct_m_p_e_synthesiser_base_a7dcd22df95ad83ee04d8ebc5ae605176}{render\+Next\+Block()}} method to produce the audio (inherited from \mbox{\hyperlink{struct_m_p_e_synthesiser_base}{MPESynthesiser\+Base}}). The voices will be started, stopped, and modulated automatically, based on the MPE/\+MIDI messages that the synthesiser receives.

Before rendering, be sure to call the \mbox{\hyperlink{class_m_p_e_synthesiser_aaa4de3655921d4971ad4334001f096b1}{set\+Current\+Playback\+Sample\+Rate()}} to tell it what the target playback rate is. This value is passed on to the voices so that they can pitch their output correctly.
