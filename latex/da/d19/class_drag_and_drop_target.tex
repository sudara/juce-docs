\doxysubsection{Detailed Description}
Components derived from this class can have things dropped onto them by a \mbox{\hyperlink{class_drag_and_drop_container}{Drag\+And\+Drop\+Container}}.

To create a component that can receive things drag-\/and-\/dropped by a \mbox{\hyperlink{class_drag_and_drop_container}{Drag\+And\+Drop\+Container}}, derive your component from this class, and make sure that it is somewhere inside a \mbox{\hyperlink{class_drag_and_drop_container}{Drag\+And\+Drop\+Container}} component.

Note\+: If all that you need to do is to respond to files being drag-\/and-\/dropped from the operating system onto your component, you don\textquotesingle{}t need any of these classes\+: instead see the \mbox{\hyperlink{class_file_drag_and_drop_target}{File\+Drag\+And\+Drop\+Target}} class.

{\footnotesize\ttfamily bool Drag\+And\+Drop\+Target\+::should\+Draw\+Drag\+Image\+When\+Over (\begin{DoxyParamCaption}{ }\end{DoxyParamCaption})\hspace{0.3cm}{\ttfamily [virtual]}}

Overriding this allows the target to tell the drag container whether to draw the drag image while the cursor is over it.

By default it returns true, but if you return false, then the normal drag image will not be shown when the cursor is over this target.
