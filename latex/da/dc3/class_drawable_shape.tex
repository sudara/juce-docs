{\footnotesize\ttfamily bool Drawable\+Shape\+::hit\+Test (\begin{DoxyParamCaption}\item[{\mbox{\hyperlink{group__juce__audio__plugin__client-_unity_ga966b05a42d98c7296097e073c627dcd0}{int}}}]{x,  }\item[{\mbox{\hyperlink{group__juce__audio__plugin__client-_unity_ga966b05a42d98c7296097e073c627dcd0}{int}}}]{y }\end{DoxyParamCaption})\hspace{0.3cm}{\ttfamily [override]}, {\ttfamily [virtual]}}

Tests whether a given point is inside the component.

Overriding this method allows you to create components which only intercept mouse-\/clicks within a user-\/defined area.

This is called to find out whether a particular x, y coordinate is considered to be inside the component or not, and is used by methods such as \mbox{\hyperlink{class_component_a6b697f6caaa753756251b4eb5ece302b}{contains()}} and \mbox{\hyperlink{class_component_a054f5a6add8f7dd6ce7cd088bc5e579f}{get\+Component\+At()}} to work out which component the mouse is clicked on.

Components with custom shapes will probably want to override it to perform some more complex hit-\/testing.

The default implementation of this method returns either true or false, depending on the value that was set by calling \mbox{\hyperlink{class_component_ac9fdcb595c1878201a641de2e1159aec}{set\+Intercepts\+Mouse\+Clicks()}} (true is the default return value).

Note that the hit-\/test region is not related to the opacity with which areas of a component are painted.

Applications should never call \mbox{\hyperlink{class_drawable_shape_ab5862ed42c7976a43791f56ae6173c27}{hit\+Test()}} directly -\/ instead use the \mbox{\hyperlink{class_component_a6b697f6caaa753756251b4eb5ece302b}{contains()}} method, because this will also test for occlusion by the component\textquotesingle{}s parent.

Note that for components on the desktop, this method will be ignored, because it\textquotesingle{}s not always possible to implement this behaviour on all platforms.

{\footnotesize\ttfamily \mbox{\hyperlink{png_8h_aa8c59027f9ab2769342f248709d68d17}{void}} Drawable\+Shape\+::paint (\begin{DoxyParamCaption}\item[{\mbox{\hyperlink{class_graphics}{Graphics}} \&}]{g }\end{DoxyParamCaption})\hspace{0.3cm}{\ttfamily [override]}, {\ttfamily [virtual]}}

Components can override this method to draw their content.

The \mbox{\hyperlink{class_drawable_shape_a9e4fb1bf71aeaa24fdc50a2a4a38c7f5}{paint()}} method gets called when a region of a component needs redrawing, either because the component\textquotesingle{}s \mbox{\hyperlink{class_component_af3ab3eabec93dd60faf983370f3a1206}{repaint()}} method has been called, or because something has happened on the screen that means a section of a window needs to be redrawn.

Any child components will draw themselves over whatever this method draws. If you need to paint over the top of your child components, you can also implement the \mbox{\hyperlink{class_component_aa15dc77c06b5a0a4192090b4a323f79c}{paint\+Over\+Children()}} method to do this.

If you want to cause a component to redraw itself, this is done asynchronously -\/ calling the \mbox{\hyperlink{class_component_af3ab3eabec93dd60faf983370f3a1206}{repaint()}} method marks a region of the component as \char`\"{}dirty\char`\"{}, and the \mbox{\hyperlink{class_drawable_shape_a9e4fb1bf71aeaa24fdc50a2a4a38c7f5}{paint()}} method will automatically be called sometime later, by the message thread, to paint any bits that need refreshing. In JUCE (and almost all modern UI frameworks), you never redraw something synchronously.

You should never need to call this method directly -\/ to take a snapshot of the component you could use \mbox{\hyperlink{class_component_a275e46e5a8147f89c142c91ba5c793a5}{create\+Component\+Snapshot()}} or \mbox{\hyperlink{class_component_a2c960bd5ec5ac76d18e82b6883096714}{paint\+Entire\+Component()}}.
