Override this method to return a custom \mbox{\hyperlink{class_accessibility_handler}{Accessibility\+Handler}} for this component.

The default implementation creates and returns a \mbox{\hyperlink{class_accessibility_handler}{Accessibility\+Handler}} object with an unspecified role, meaning that it will be visible to accessibility clients but without a specific role, action callbacks or interfaces. To control how accessibility clients see and interact with your component subclass \mbox{\hyperlink{class_accessibility_handler}{Accessibility\+Handler}}, implement the desired behaviours, and return an instance of it from this method in your component subclass.

The accessibility handler you return here is guaranteed to be destroyed before its \mbox{\hyperlink{class_component}{Component}}, so it\textquotesingle{}s safe to store and use a reference back to the \mbox{\hyperlink{class_component}{Component}} inside the \mbox{\hyperlink{class_accessibility_handler}{Accessibility\+Handler}} if necessary.

{\footnotesize\ttfamily \mbox{\hyperlink{png_8h_aa8c59027f9ab2769342f248709d68d17}{void}} juce\+::\+Popup\+Menu\+::\+Helper\+Classes\+::\+Header\+Item\+Component\+::paint (\begin{DoxyParamCaption}\item[{\mbox{\hyperlink{class_graphics}{Graphics}} \&}]{g }\end{DoxyParamCaption})\hspace{0.3cm}{\ttfamily [inline]}, {\ttfamily [override]}, {\ttfamily [virtual]}}

Components can override this method to draw their content.

The \mbox{\hyperlink{structjuce_1_1_popup_menu_1_1_helper_classes_1_1_header_item_component_a8d9beaa1a6cc6e6c3c9215b0dc6c660e}{paint()}} method gets called when a region of a component needs redrawing, either because the component\textquotesingle{}s \mbox{\hyperlink{class_component_af3ab3eabec93dd60faf983370f3a1206}{repaint()}} method has been called, or because something has happened on the screen that means a section of a window needs to be redrawn.

Any child components will draw themselves over whatever this method draws. If you need to paint over the top of your child components, you can also implement the \mbox{\hyperlink{class_component_aa15dc77c06b5a0a4192090b4a323f79c}{paint\+Over\+Children()}} method to do this.

If you want to cause a component to redraw itself, this is done asynchronously -\/ calling the \mbox{\hyperlink{class_component_af3ab3eabec93dd60faf983370f3a1206}{repaint()}} method marks a region of the component as \char`\"{}dirty\char`\"{}, and the \mbox{\hyperlink{structjuce_1_1_popup_menu_1_1_helper_classes_1_1_header_item_component_a8d9beaa1a6cc6e6c3c9215b0dc6c660e}{paint()}} method will automatically be called sometime later, by the message thread, to paint any bits that need refreshing. In JUCE (and almost all modern UI frameworks), you never redraw something synchronously.

You should never need to call this method directly -\/ to take a snapshot of the component you could use \mbox{\hyperlink{class_component_a275e46e5a8147f89c142c91ba5c793a5}{create\+Component\+Snapshot()}} or \mbox{\hyperlink{class_component_a2c960bd5ec5ac76d18e82b6883096714}{paint\+Entire\+Component()}}.
