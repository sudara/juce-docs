\doxysubsection{Detailed Description}
Extended \mbox{\hyperlink{class_steinberg_1_1_vst_1_1_i_audio_processor}{IAudio\+Processor}} interface for a component\+: \mbox{\hyperlink{class_steinberg_1_1_vst_1_1_i_process_context_requirements}{Vst\+::\+IProcess\+Context\+Requirements}}


\begin{DoxyItemize}
\item \mbox{[}plug imp\mbox{]}
\item \mbox{[}extends \mbox{\hyperlink{class_steinberg_1_1_vst_1_1_i_audio_processor}{IAudio\+Processor}}\mbox{]}
\item \mbox{[}released\+: 3.\+7.\+0\mbox{]}
\item \mbox{[}mandatory\mbox{]}
\end{DoxyItemize}

To get accurate process context information (\mbox{\hyperlink{struct_steinberg_1_1_vst_1_1_process_context}{Vst\+::\+Process\+Context}}), it is now required to implement this interface and return the desired bit mask of flags which your audio effect needs. If you do not implement this interface, you may not get any information at all of the process function .

The host asks for this information once between initialize and set\+Active. It cannot be changed afterwards.

This gives the host the opportunity to better optimize the audio process graph when it knows which plug-\/ins need which information.

Plug-\/\+Ins built with an earlier SDK version (\texorpdfstring{$<$}{<} 3.\+7) will still get the old information, but the information may not be as accurate as when using this interface.
