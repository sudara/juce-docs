Components can override this method to draw their content.

The \mbox{\hyperlink{classjuce_1_1_colour_selector_1_1_swatch_component_a4e03bfea1b64be2f77943ecfc4c0f002}{paint()}} method gets called when a region of a component needs redrawing, either because the component\textquotesingle{}s \mbox{\hyperlink{class_component_af3ab3eabec93dd60faf983370f3a1206}{repaint()}} method has been called, or because something has happened on the screen that means a section of a window needs to be redrawn.

Any child components will draw themselves over whatever this method draws. If you need to paint over the top of your child components, you can also implement the \mbox{\hyperlink{class_component_aa15dc77c06b5a0a4192090b4a323f79c}{paint\+Over\+Children()}} method to do this.

If you want to cause a component to redraw itself, this is done asynchronously -\/ calling the \mbox{\hyperlink{class_component_af3ab3eabec93dd60faf983370f3a1206}{repaint()}} method marks a region of the component as \char`\"{}dirty\char`\"{}, and the \mbox{\hyperlink{classjuce_1_1_colour_selector_1_1_swatch_component_a4e03bfea1b64be2f77943ecfc4c0f002}{paint()}} method will automatically be called sometime later, by the message thread, to paint any bits that need refreshing. In JUCE (and almost all modern UI frameworks), you never redraw something synchronously.

You should never need to call this method directly -\/ to take a snapshot of the component you could use \mbox{\hyperlink{class_component_a275e46e5a8147f89c142c91ba5c793a5}{create\+Component\+Snapshot()}} or \mbox{\hyperlink{class_component_a2c960bd5ec5ac76d18e82b6883096714}{paint\+Entire\+Component()}}.
