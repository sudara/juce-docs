\doxysubsection{Detailed Description}
MIDI Learn interface\+: \mbox{\hyperlink{class_steinberg_1_1_vst_1_1_i_midi_learn}{Vst\+::\+IMidi\+Learn}}


\begin{DoxyItemize}
\item \mbox{[}plug imp\mbox{]}
\item \mbox{[}extends \mbox{\hyperlink{class_steinberg_1_1_vst_1_1_i_edit_controller}{IEdit\+Controller}}\mbox{]}
\item \mbox{[}released\+: 3.\+6.\+12\mbox{]}
\item \mbox{[}optional\mbox{]}
\end{DoxyItemize}

If this interface is implemented by the edit controller, the host will call this method whenever there is live MIDI-\/\+CC input for the plug-\/in. This way, the plug-\/in can change its MIDI-\/\+CC parameter mapping and inform the host via the \mbox{\hyperlink{class_steinberg_1_1_vst_1_1_i_component_handler_a65e162e3686dbcd5f2739b6481f69186}{IComponent\+Handler\+::restart\+Component}} with the k\+Midi\+CCAssignment\+Changed flag. Use this if you want to implement custom MIDI-\/\+Learn functionality in your plug-\/in.
